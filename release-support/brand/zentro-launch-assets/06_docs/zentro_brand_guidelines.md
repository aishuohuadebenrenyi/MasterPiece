## 2. Logo 基础规范

### 2.1 版本定义

| 版本 | 用途 | 文件位置 |
|---|---|---|
| 完整横版 Logo | 官网、公众号头图、活动海报、商店详情页 | `01_logo_base/png/logo_full_transparent_original_crop.png` |
| 图形符号 Icon | App 图标、小程序头像、角标、水印、社交头像 | `01_logo_base/png/logo_symbol_transparent_original_crop.png` |
| 深蓝单色版 | 印刷、低成本物料、单色限制场景 | `01_logo_base/png/logo_symbol_navy_transparent.png` |
| 反白版 | 深色背景、夜间海报、舞台相关视觉 | `01_logo_base/png/logo_symbol_white_transparent.png` |
| 渐变版 | 默认品牌展示、宣传视觉主图 | `01_logo_base/png/logo_symbol_gradient_rebuilt.png` |

### 2.2 安全边距

- 图形符号四周最小安全边距：画布短边的 12%。
- App / 小程序图标核心图形建议控制在画布宽度的 72% - 78%，避免被圆形或圆角遮罩裁切。
- 横版 Logo 左右安全边距：至少等于英文 `Zentro` 字高的 0.5 倍。
- 任何背景图、广告图、海报中，Logo 不应贴边、不应被二维码、按钮、活动照片遮挡。

### 2.3 最小应用尺寸

| 场景 | 最小尺寸 |
|---|---|
| 图形符号 - 屏幕 | 24px，不建议再小 |
| 图形符号 - 印刷 | 8mm 宽 |
| 横版 Logo - 屏幕 | 120px 宽 |
| 横版 Logo - 印刷 | 25mm 宽 |
| 角标 / 水印 | 建议宽度为画面短边的 8% - 14%，透明度 55% - 75% |

### 2.4 禁止用法

- 不要横向或纵向拉伸 Logo。
- 不要改变渐变顺序，默认方向为橙色 `#FF7A18` → 玫红 `#D946EF` → 蓝色 `#2563EB`。
- 不要在复杂照片上直接放置彩色 Logo；复杂背景应使用白色底板或反白版。
- 不要在 App Store / Google Play 图标原始上传图里加入“官方”“NEW”“免费”“第一”等平台敏感或误导性标识。
- 不要把小程序码/二维码占位模板直接投放；必须替换为真实可扫码码图。

## 4. 场景使用说明

### 4.1 应用上架

- 微信小程序头像使用 `mini_program_avatar_144x144.png` 或更高清的 `512x512` 版本，核心图形已预留圆形裁切安全区。
- iOS 图标上传优先使用 `app_store_icon_1024x1024_square_no_transparency.png`，圆角版仅用于预览。
- Google Play 图标使用 `google_play_icon_512x512.png`；不加阴影与圆角，由平台统一处理。
- 截图模板目前为品牌占位版，正式上架前需要替换为真实产品界面截图。

### 4.2 官方阵地运营

- 全平台头像优先保持同一图形符号，不建议频繁更换。
- 公众号/小红书/抖音背景图可直接替换文案，但 Logo 位置建议保持不变。
- 日常内容模板中的角标建议固定在左下或右下，避免与平台按钮、标题、评论区 UI 冲突。

### 4.3 流量投放

- 投放基底已预留产品截图 / 场景图区域，可替换为真实 App 截图或线下活动照片。
- 避免使用夸张承诺、绝对化词、排名词、价格诱导词。
- 开屏广告模板为无文字居中 Logo，适合作为品牌曝光基础版。
- 落地页头部组件可作为 H5 / 小程序落地页首屏品牌区。

### 4.4 线下推广

- A4/A3 PNG 已按 300DPI 导出，可用于普通宣传单和海报初稿。
- 易拉宝使用 SVG 矢量模板交给印厂或设计师二次排版；大幅面印刷建议优先使用矢量源文件。
- `zentro_symbol_ai_compatible.ai` 是 AI 兼容 EPS 结构，可用 Illustrator 打开后另存为原生 `.ai`。

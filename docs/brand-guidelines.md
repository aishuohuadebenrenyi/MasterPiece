# Zentro｜即兴一下品牌视觉规范

更新时间：2026-06-25

本文以 2026-06-23 提交的合成预览和 2026-06-24 `release-support/brand/zentro-launch-assets/` 正式源文件为目标品牌基准。原始资产包包含 159 个已验收条目，PNG 尺寸/可打开性及矢量、文档、CSV 非空检查全部通过；Logo 颜色和几何以矢量源文件为最高事实源。当前工程中的“即兴工具箱”“即兴现场”属于待迁移口径；在完成产品命名、账号主体和上架材料确认前，不应在代码中批量替换名称。

## 1. 核心识别

品牌调性为：**灵感闪现、无限延展、流动连接、年轻创造力**。

- 无限曲线：表达即兴创作没有固定终点，灵感可以持续流动。
- 左上短线：表达灵感被触发的瞬间；不单独作为功能图标。
- 右下圆点：表达一次即兴被记录、沉淀并形成价值。
- 橙—粉—紫—蓝渐变：从能量启动过渡到想象、连接和可信赖的工具感。
- 轻量 3D 几何：只用于宣传背景和品牌氛围，不进入高频操作控件。

产品端必须保留清楚的信息层级和临场效率，不因宣传物料的渐变、光晕或 3D 元素增加操作噪声。

## 2. 品牌色

CMYK 为未绑定印刷 ICC 配置时的通用近似换算值，只用于前期沟通；正式印刷必须打样确认。

### 2.3 色彩心理与场景适配

- 橙黄负责触发“马上行动”的年轻创作感，但普通字号按钮使用深墨文字，不使用白字硬扛对比度。
- 大面积背景和卡片采用近白与纯白，避免灰绿或米绿让工具显得沉闷。
- 灰蓝提供信息层级；工具蓝承担需要可读性的链接与选中导航。
- 活力绿只用于成功、进度和正向点缀，不抢主操作。
- 现场主题继续保留深墨和紫蓝，确保低光环境下的任务聚焦，不与灵感主题混合。

### 2.4 渐变规则

- Logo 母版渐变固定为 `#FF7A18 → #D946EF → #2563EB`，矢量停靠点为 0% / 48% / 100%。
- 灵感主题的按钮、Tab、chip 和工具操作以橙黄平面色或橙黄浅底表达，不使用橙粉紫蓝宣传渐变。
- 页面背景仅允许近白暖灰到极浅暖黄/浅青灰的低对比过渡；卡片和输入表面保持高明度平面色。
- 现场主题可继续使用既有紫蓝局部渐变，但单个控件最多使用相邻两色。
- 禁止给正文、长列表背景、错误状态和数据图表主数据使用装饰渐变。

## 3. 双主题规范

设置页最终只展示两套正式主题。持久化值继续使用 `default` / `vivid`，分别映射为“灵感”与“现场”，避免迁移用户偏好。

### 3.1 核心语义 token

| 语义 | 灵感 `default` | 现场 `vivid` | 规则 |
| --- | --- | --- | --- |
| 页面底色 | `#FAFAF7 → #F7F8F3 → #F3F7F5` | `#111827` | 灵感使用近白暖灰到极浅暖黄/浅青灰；现场不使用纯黑 |
| 次级底色 | `#FFF5E7` | `#17162B` | 只作浅橙氛围，不形成大面积色块 |
| 卡片 | `#FFFFFF` | `#211D3A` | 与页面底色保持可见层级 |
| 强卡片/弹层 | `#FFFFFF` | `#2A2448` | 任务型内容优先稳定可读 |
| 输入底色 | `#F5F7F3` | `#2F294F` | 聚焦态用品牌边界，不改变布局 |
| 主文字 | `#2F3337` | `#F7F8FA` | 对比度必须达到 4.5:1 |
| 正文文字 | `#65717D` | `#D4D8E3` | 灵感正文保持清晰灰蓝层级 |
| 辅助文字 | `#8D9AA5` | `#AEB6C7` | 不能用透明度制造不可读灰字 |
| 边界 | `rgba(63,127,168,0.13)` | `rgba(255,255,255,0.14)` | 卡片不依赖重阴影分层 |
| 品牌主色 | `#FF9F1C` | `#8A26FF` | 仅一个页面级主动作 |
| AA 主按钮 | `#FF9F1C` + `#3A2A10` | `#8A26FF` + 白字 | 灵感使用深棕文字；现场约 5.52:1 |
| 次级信息 | `#2F6F93` | `#0066E6` | 灵感链接和 Tab 选中使用工具蓝 |
| 灵感强调 | `#7BD89A` | `#FF3D71` | 灵感主题仅作成功、进度和正向点缀 |
| 成功 | `#267A4E` | `#49D3B4` | 不只依赖颜色表达结果 |
| 警告 | `#A85C00` | `#FFB45C` | 同时提供文字或图标 |
| 错误 | `#C23B4A` | `#FF7185` | 危险操作与品牌粉分离语义 |
| 遮罩 | `rgba(17,24,39,0.36)` | `rgba(0,0,0,0.68)` | 任务弹层仍保持清晰焦点 |

`#FF9F1C` 不用于普通字号白字按钮；灵感主按钮搭配 `#3A2A10` 深棕文字。`#8D9AA5` 只用于辅助文字和低强调图标，不承载关键正文。灵感主题链接和 Tab 文字使用 `#2F6F93`。`#8A26FF` 配白字约 5.52:1。

### 3.2 主题状态

| 状态 | 规范 |
| --- | --- |
| normal | 使用对应主题的语义表面和文字，不在页面局部改品牌色 |
| pressed | `transform: scale(0.98)`；颜色只降低一档明度，持续 160ms |
| selected | 使用品牌 tint、明确边界和文字变化，不只靠阴影 |
| disabled | 保留可读标签，整体不透明度 48%，移除阴影并禁止事件 |
| loading | 保留按钮尺寸与原文语义，显示进度并禁止重复提交 |
| success/warning/error | 同时使用文字/图标，不以绿、橙、红单独传达 |
| overlay | 背景不可操作；任务型弹层禁止点遮罩关闭 |

当前 `theme-alt` 不对用户开放，标记为废弃候选；后续实施阶段确认无引用后移除，不形成第三套事实源。

## 4. Logo 使用

- 优先使用彩色 Logo；单色环境使用官方黑色或反白版本。
- 图形符号四周最小安全边距为画布短边的 12%；App/小程序图标核心图形控制在画布宽度的 72%–78%。
- 横版 Logo 左右安全边距至少为英文 `Zentro` 字高的 0.5 倍。
- 图形符号屏幕最小 24px、印刷最小 8mm；横版 Logo 屏幕最小宽 120px、印刷最小宽 25mm。
- 角标/水印建议占画面短边 8%–14%，透明度 55%–75%。
- 深色背景使用反白字标或带足够留白的彩色图形，不给 Logo 添加描边、阴影或外发光。
- 禁止改变长宽比、重排短线/圆点、旋转、裁切、换渐变顺序或把 Logo 当作普通功能图标。

## 5. 字体与排版

- 中文：思源黑体；小程序系统回退为 PingFang SC、Microsoft YaHei、sans-serif。
- 英文与数字：Poppins；不可用时回退系统无衬线字体。
- 页面标题 56rpx/900，区块标题 40rpx/800，卡片标题 32rpx/800，列表标题 28–30rpx/700，正文 26rpx/500–600，辅助文字 24rpx/500，标签 22–24rpx/700。
- 正文行高不低于 1.5；宣传标题可以更紧，产品端不使用极细字重。

## 6. 应用边界

- 产品端优先效率、对比度和状态清晰；宣传端可以强化渐变、3D 和品牌氛围。
- 一个页面只允许一个视觉主动作；品牌橙、紫、蓝不能并列成为三个主按钮。
- 主题切换只替换语义 token，不复制页面或组件样式。
- 新增颜色必须先证明现有语义无法表达，并同步本规范和主题 token。
- 品牌命名迁移需单独核对小程序名称、隐私政策、CloudBase 文案、分享物料和应用商店材料，不能仅替换界面标题。

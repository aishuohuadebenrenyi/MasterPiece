# 全产品提示文案审查清单

更新时间：2026-06-24

本文档审查即兴工具箱当前 13 个已注册页面、共用组件及运行时提示文案，目标是减少重复解释和教学式表达，让产品更简洁、克制、易扫读。本轮只记录审查结论和推荐文案，不修改页面实现、接口、字段规则或交互行为。

## 1. 审查基线

- 页面范围以 `wechat-cloudbase-app/frontend/app.json` 为准，共 13 个页面。
- 文案事实以当前工作区的 WXML、TS、JS 为准，包含未提交修改，不按 Git 旧版本推断。
- 用户提供的设置页反馈弹层截图作为重复提示问题的视觉样本；其余结论来自代码中的静态文案、动态空态、Toast 和 Modal 扫描。
- 帮助页和隐私政策同样执行减法，但隐私政策不得因精简而改变真实数据处理方式或遗漏必要信息。

## 2. 统一判断规则

1. **一项信息只出现一次**：标签回答“填什么”，placeholder 只给短示例，辅助说明只保留无法从控件理解的规则。
2. **先动作，后解释**：操作页优先展示当前任务和下一步，不在首屏重复讲产品价值。
3. **空态最多三层**：短标题、一句说明、一个主动作；次动作只在存在明确第二条路径时保留。
4. **状态反馈说结果**：成功 Toast 使用“已保存”“已删除”等结果短句；失败提示说明失败对象并给出“重试”等下一步。
5. **危险操作不省略后果**：删除、注销等操作保留影响范围和不可恢复提示。
6. **帮助页回答任务**：不逐字段复述产品结构，只保留完成任务必须知道的规则。
7. **隐私政策简短但完整**：必须保留收集范围、使用目的、存储与隔离、安全措施、用户权利、未成年人保护、政策更新和联系方式。

优先级定义：

- **P0**：同屏重复明显、影响填写或阅读效率，优先改。
- **P1**：页面信息密度偏高或状态文案冗长，随下一轮体验整改处理。
- **P2**：不阻碍使用，主要用于统一语气和细节。

## 5. 运行时提示与确认框

### 5.1 Toast 统一词表

| 类型 | 推荐格式 | 示例 |
| --- | --- | --- |
| 保存成功 | `已保存{对象}` | `已保存灵感`、`已保存练习记录` |
| 创建成功 | `已加入{位置}` | `已加入素材库`、`已加入排练` |
| 更新成功 | `已更新{对象}` | `已更新排练计划` |
| 删除成功 | `已删除` | 对象在当前页面已明确时不重复名称 |
| 加载失败 | `{对象}加载失败，请重试` | `素材加载失败，请重试` |
| 写入失败 | `{动作}失败，请重试` | `保存失败，请重试`、`提交失败，请重试` |
| 必填校验 | 直接说明缺少内容 | `请选择反馈类型`、`请填写反馈内容` |

需要优先收敛的变体：

- `保存失败，请稍后再试`、`保存失败，请稍后重试` 统一为 `保存失败，请重试`。
- `加载素材失败`、`当前还没有可显示的素材` 按真实状态分别使用失败态和空态，不在 Toast 中重复空态说明。
- `反馈内容需为 10–500 字` 保留为提交校验；表单静态说明不再重复相同限制。
- `联系方式不能超过 100 字` 只在超限时出现，不常驻表单。

### 5.2 Modal 规则

- 删除确认统一说明“删除对象 + 是否可恢复”，不添加道歉或劝阻性套话。
- 账号注销必须保留删除范围和不可恢复说明，不改成通用的“确定吗”。
- 当前练习记录、排练记录的 `删除后不可恢复` 明确且必要，保留。
- 素材删除是否不可恢复需以服务端真实行为为准；实施文案前先核对，不在本审查中新增承诺。

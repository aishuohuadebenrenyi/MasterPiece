# 即兴工具箱上线前最终核查报告

核查日期：2026-06-25  
核查对象：`wechat-cloudbase-app/` 当前工作树  
核查结论：**No-Go，暂不满足正式发布条件**

## 1. 结论摘要

当前工程的技术选型、基础分层、公共组件骨架和核心复合写入方案总体合理，前端体积约 1 MB，已启用 Skyline、Glass-Easel、按需组件加载和构建压缩。`syntax-check`、`typecheck` 通过，页面与组件事件静态绑定完整，前端业务请求统一经过服务层，数据库规则文件禁止客户端直接读写，练习完成和排练完成采用事务。

但当前不能判定为“已准备上线”。本轮确认 6 个 P0 发布阻断项、9 个 P1 高风险项和 4 个 P2 改进项。最关键的问题是：

1. 发布环境 AppID、三套 CloudBase envId、云函数和数据库规则实际部署均无可审计证据。
2. `contract-check` 当前失败，既定发布门禁未通过。
3. 素材详情没有按 ID 查询能力，只回拉默认前 50 条素材；搜索、筛选、分页结果或分享深链中的后续素材可能打开“未找到”。
4. 隐私政策未披露实际处理的昵称、剧团名和头像文件，声明与实现不一致。
5. 素材、练习、排练、反馈、注销等真实 CloudBase 成功链路未完成本轮运行态验收。
6. 微信开发者工具窗口无法被本轮自动化通道捕获，15 张运行态截图和真机证据仍缺失。

正式发布前必须关闭全部 P0；P1 必须修复或由负责人书面接受残余风险，并完成回归。

## 2. 核查范围与证据边界

### 2.1 已核查

- 13 个注册页面、3 个主 Tab、全部页面跳转目标。
- 14 个 `frontend/components/` 公共组件和自定义 TabBar。
- 9 个前端业务服务模块、轻量 store、领域类型和统一云调用层。
- `improv-api` 的 29 个 action、字段白名单、所有权过滤、事务和账号删除逻辑。
- 8 个 `improv_` 集合、数据库规则文件、工程配置和依赖锁定。
- 核心链路：素材、灵感、练习、排练、方法卡、资料、反馈和账号注销。
- 帮助、隐私、错误提示、空态、下拉刷新、主题和分享入口。

### 2.2 本轮无法证明

- 真实 AppID、release envId 和 CloudBase 环境绑定。
- 云函数、素材数据、索引、数据库安全规则是否已部署到正式环境。
- 真机中文输入、安全区、弱网、断网恢复、后台切换和低端机帧率。
- 微信审核后台隐私声明、用户隐私保护指引和代码实际调用是否一致。
- 读屏、焦点、动态字体、色觉和完整 WCAG 合规。
- 生产依赖在线漏洞审计：根工程无生产依赖且审计通过；云函数依赖审计联网请求持续无结果，本轮未形成有效结论。

微信开发者工具处于运行状态，但自动化接口两次无法返回窗口，系统截图脚本也未捕获到开发者工具画面。本报告不使用旧图或占位图替代本轮证据。

## 3. 风险分级

- **P0 / 阻断**：可能导致发布失败、核心功能不可用、隐私声明不一致或无法证明生产环境安全，发布前必须关闭。
- **P1 / 高**：可能造成数据异常、部分流程失败、明显体验障碍或较高维护/扩展风险；应在发布前修复或书面接受风险。
- **P2 / 中低**：不直接阻断首发，但会增加维护成本、回归概率或体验不一致。

## 5. 四类核查结论

### 5.1 前后端架构

健康项：

- 原生小程序 + Skyline + Glass-Easel 与当前产品规模匹配，没有迁移大型框架的必要。
- 3 个 Tab 和 13 个页面路由目标均存在，子页面职责基本清晰。
- `lazyCodeLoading: requiredComponents`、构建压缩和约 1 MB 前端体积对首发有利。
- 页面不直接散落 `wx.cloud.callFunction`，统一经过 `services/cloud.ts`；具备 requestId、12 秒超时、loading 和用户可理解的错误转换。
- 聚合云函数适合当前 MVP，复合练习/排练写入已经使用事务。

主要隐患：

- 缺少按 ID 查询导致分页、筛选和深链断裂。
- 500 条内存扫描是明确扩展上限，不是长期分页架构。
- 云函数资源与发布回滚参数未版本化，生产稳定性无法仅靠代码证明。
- AppID/envId/规则/索引/数据版本都需要环境级证据。

### 5.2 组件体系

健康项：

- 14 个公共组件均有实际调用，除 1 个页面声明外没有发现孤儿组件。
- WXML 中所有事件处理函数均能在对应脚本找到；公共组件触发的自定义事件均有实现。
- `bottom-sheet`、`empty-state-panel`、`form-field`、`filter-section`、`selection-card`、`record-card`、`subpage-header` 已覆盖主要重复结构。
- `bindinput` 扫描为 0，符合 Skyline 中文输入约束。

主要隐患：

- 三个主页面体积过大，回归面集中。
- 列表刷新、可访问状态和双主题状态仍未统一形成运行门禁。
- `record` 存在一个未使用组件声明。

当前不建议为了“组件化率”在发布前拆大页面；先修真实 P0/P1，再以测试保护拆分。

### 5.4 用户体验

代码级优点：核心记录链路保持“入口 → 记录 → 保存”；写入失败通常保留草稿；空态、错误态、重试文案和任务型弹层已有统一基础；隐私、帮助、反馈和注销入口可找到。

主要体验风险：

- 后 50 条素材详情失败会让用户把“可见但打不开”理解为数据丢失。
- 帮助页对类型、能力、场景、标签边界说明不足，用户可能把自由标签误认为一级分类。
- 我的页信息密度高，发现/记录/我的三页状态复杂，需用真实数据态截图判断首屏主次。
- 历史列表下拉刷新实现不一致，用户可能无法判断数据是否更新。
- 读屏、非手势替代、错误焦点和 selected 状态尚未形成证据。

2. 部署并验证数据库安全规则。
3. 对越权、非法字段、超长输入和注销部分失败做负向测试。
4. 在联网 CI 完成云函数生产依赖审计。

### Gate C：真实环境闭环

1. 绑定体验/正式环境并部署同一云函数版本。
2. 导入 163 条素材，验证第 1/50/51/163 条详情和分享冷启动。
3. 完成素材、灵感、练习、排练、反馈、资料和注销编号用例。
4. 检查数据库记录、requestId、错误日志和重试行为。

### Gate D：体验与审核

1. 补齐 15 张双主题/多状态截图。
2. 在至少一台 iOS 和一台 Android 真机验证中文输入、安全区、滚动、刷新、后台恢复和弱网。
3. 完成读屏与非手势路径检查。
4. 对微信版本、隐私弹窗、分享卡片、服务类目、名称/Logo/截图和审核文案做最终复核。

## 7. 最终判定条件

只有同时满足以下条件，才能把结论从 No-Go 改为 Go：

- 6 个 P0 全部关闭并附证据。
- 三项本地门禁全绿，关键后端测试进入 CI。
- 正式环境的 AppID、envId、云函数、集合、索引、规则和素材版本一致。
- 主要成功/失败链路完成真机验收，没有未解释的数据差异。
- 隐私政策、后台指引和实际字段/API 完全一致。
- 发布负责人完成截图、日志、数据记录和回滚方案签字。

在这些证据形成前，可以继续进行开发者工具调试或体验版测试，但不应描述为“已准备正式上线”。

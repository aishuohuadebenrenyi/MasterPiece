## 3. 数据库安全规则验收

正向验证：

1. 通过 `improv-api` 调用 `material.list` 成功。
2. 通过 `improv-api` 创建一条测试反馈成功。

反向验证：

1. 小程序端直接读取任一私有集合应被拒绝。
2. 小程序端直接写入任一 `improv_` 集合应被拒绝。

记录要求：

- 环境
- 时间
- 操作者
- requestId
- 控制台截图或日志

## 5. 隐私与审核核对

微信后台隐私指引必须覆盖以下实际字段和能力：

- OpenID：用户识别和数据隔离。
- 昵称 `displayName`：用户主动填写，用于个人资料展示。
- 剧团名 `troupeName`：用户主动填写，用于个人资料展示。
- 头像：用户主动调用相册/相机选择，上传至 CloudBase 云存储。
- 用户主动创建的灵感、排练、练习复盘、方法卡、自定义素材。
- 收藏、练过、最近练习等素材状态。
- 反馈内容、选填联系方式、来源页面和版本。

页面隐私政策、微信后台隐私指引和实际 API/字段必须逐项一致。

## 6. 依赖安全审计

在可联网 CI 或受控本机执行：

```bash
cd wechat-cloudbase-app/backend/cloudfunctions/improv-api
npm audit --omit=dev
```

关闭标准：

- 有可追溯审计日志。
- 无未接受的 high / critical 漏洞。
- 如果漏洞来自 `wx-server-sdk` 上游依赖且无法安全升级，需记录风险接受人、影响范围和后续跟进日期。

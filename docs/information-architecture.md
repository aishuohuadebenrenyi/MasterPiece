# 信息架构与页面链路

更新时间：2026-06-08

## 1. 导航结构

当前小程序采用 3 个主 Tab：

- 发现：游戏发现、筛选、随机抽卡、添加游戏、进入游戏详情。
- 记录：语音速记、灵感记录、快速开启排练、今日记录、暂停排练。
- 我的：待整理、个人沉淀、灵感记录、排练记录、游戏记录。

独立“计划”Tab 不再保留。排练计划和排练过程都归入记录链路。

## 3. 弹层矩阵

当前产品使用半弹窗和悬浮卡片承载轻量交互。半弹窗分为轻量半弹窗和任务型半弹窗：

- 筛选游戏：人数、时长、目标、状态；每个分类独占一行，选项在该分类行内排列。
- 条件抽卡：抽中游戏、再抽一张、进入详情。
- 添加游戏：名称、分类 chips、人数、时长、简短描述；玩法步骤默认作为补充项，不阻断保存，当前实现保留底部操作区并将按钮组居中显示。
- 语音速记：录音计时、归类、摘要预览。
- 快速开始游戏：搜索游戏、默认展示可选游戏列表、选择后开始单次游戏。
- 快速开启排练：团队、时长、目标、添加游戏方式。
- 关联选择：选择关联游戏或关联排练。
- 添加到排练：搜索游戏并加入排练。
- 今日记录：查看今日灵感和今日排练。
- 个人沉淀/灵感列表：在我的页查看列表。
- 记录详情悬浮卡：查看单条沉淀或灵感详情。

## 4. 页面与弹层使用规则

- 主 Tab 承载高频入口和聚合状态，不承载过长表单。
- 子页面承载深读、完整编辑和排练过程。
- 轻量半弹窗承载筛选、抽卡、关联选择和语音速记，建议高度控制在 50-65vh。
- 任务型半弹窗承载快速添加游戏、快速开启排练等单一临场任务，建议高度控制在 70-82vh。
- 超过一个主任务、需要长表单或完整编辑时，应进入子页面。
- 任务型半弹窗的主按钮不强制固定在独立底栏；当独立底栏造成遮挡或视觉断层时，应回到内容流末尾。
- 悬浮卡片承载当前页面内的只读详情，不再额外增加跳转深度。
- 临场记录链路应尽量控制在入口 -> 记录 -> 保存。

## 5. 关键链路

### 5.1 找游戏

发现页 -> 搜索/筛选/标签 -> 游戏卡片 -> 游戏详情 -> 开始游戏。

详情页用于深读判断和启动单次游戏，发现页卡片和随机抽卡不直接进入游戏实践反馈。

当前阶段补充：

- 无游戏库时，发现页不显示完整发现页骨架，先进入单一大卡片空态。
- 该空态的唯一主动作是 `添加游戏`，点击后打开当前已有的添加游戏底部弹层。
- 一旦存在至少一个游戏，发现页恢复为正常的搜索、筛选、浏览与记录结构。

### 5.2 游戏实践反馈

游戏详情页点击开始游戏 -> 结束并复盘 -> 游戏实践反馈页 -> 保存反馈 -> 可选择沉淀为方法卡。

目标体验是保存直接完成，沉淀作为保存后的二级动作。

补充边界：

- 历史排练保持只读，不允许从游戏反馈页回写或追加编辑。
- 游戏、灵感、排练复盘的后续归档统一走“我的 -> 待整理”。

### 5.3 灵感记录

记录页 -> 语音速记或手动记录 -> 灵感草稿/灵感编辑页 -> 保存灵感 -> 可沉淀为方法卡。

灵感强调“先别忘”，不应强制一开始就补齐所有结构化字段。

当前开发阶段补充：

- 记录页“今日灵感 / 今日排练”统计默认以云端聚合结果和当前会话内存态为准。
- 云端成功返回空结果时，记录页应显示空态或 0，不回退旧本地缓存数据。

### 5.4 排练过程

记录页 -> 快速开启排练 -> 排练记录页 -> 添加游戏、切换状态、记录 Keep / Try -> 暂停或结束复盘。

排练记录页主要处理过程态，不承载完整整体复盘。

### 5.5 排练复盘

排练记录页 -> 排练复盘页 -> 保存整体 Keep / Try / 下次提醒 -> 完成。

复盘保存后应尽量直接完成，不再强制多次确认。

### 5.6 资产回看

我的页 -> 待整理 -> 悬浮详情卡 -> 沉淀为方法卡。

我的页 -> 个人沉淀 -> 来源筛选 -> 悬浮详情卡。

我的页 -> 灵感记录列表 -> 悬浮详情卡。

我的页 -> 团队排练记录 -> 当前页只读排练详情。

当前开发阶段补充：

- 我的页、团队排练记录页和游戏记录页默认以云端正式数据为准。
- 云端成功返回空数组时，应进入空态；只有请求失败时才显示错误态或当前会话内待同步数据。
- 回看类列表页在空数据时不保留 `0 条` 摘要卡，而是直接进入带下一步动作的空态。
- 首次进入 `我的` 页且没有个人数据时，先展示个人空间的价值说明和起步动作，而不是默认完整资料页。
- 当方法卡、灵感和排练记录开始累积后，`我的` 页再逐步恢复为资产回看页结构。
- 我的页顶部个人卡片支持编辑名字和头像；资料默认走云端个人资料记录，标签固定为 `即兴主理人` 和 `个人空间`。
- 如果填写了 `troupeName`，顶部标签改为 `剧团名 + 个人空间`。
- 我的页“待整理”聚合可沉淀来源，不自动生成方法卡，用户进入详情后手动沉淀。
- 个人沉淀列表按来源筛选，来源包括灵感、游戏实践和排练复盘。

## 6. 编辑态与查看态

当前排练记录页承担过程编辑；团队历史回看已先收敛到 `team-records` 当前页的只读详情态。后续如果只读详情继续变复杂，再拆独立详情页。

- 排练过程页：可编辑、可暂停、可结束。
- 当前排练详情：`team-records` 页内只读回看。
- 后续规划：如果需要复制为新排练、深链接直达或承载更多详情，再拆独立排练详情页。

团队历史回看不应默认进入可编辑排练过程页，避免查看历史时误操作。

## 7. 深度控制目标

目标链路：

- 发现 -> 详情 -> 开始游戏 -> 结束复盘 -> 保存反馈。
- 发现 -> 抽卡 -> 详情 -> 开始游戏 -> 结束复盘 -> 保存反馈。
- 记录 -> 语音 -> 自动保存灵感草稿。
- 排练记录 -> 结束复盘 -> 保存完成。

避免链路：

- 发现 -> 详情 -> 反馈 -> 确认 -> 方法卡。
- 记录 -> 语音 -> 灵感页 -> 多字段填写 -> 保存前确认。
- 排练记录 -> 复盘 -> 确认 -> 方法卡。

## 8. 当前阶段数据展示约定

- 当前功能开发和页面交互开发阶段，不启用 `store` 本地持久化缓存。
- 页面只使用：
  - 云端真实数据
  - 当前会话内存态
  - 空态 / 错误态
- 如果之前运行过旧版本，验证空态前需要先清理微信开发者工具 Storage / 清缓存 / 编译缓存。

# 即兴工具箱产品文档

版本：MVP 文档版
更新时间：2026-06-07
当前实现：`wechat-cloudbase-app/`
当前原型参考：`prototype_complete/index.html`

## 1. 产品定位

即兴工具箱是一款面向即兴主理人、排练带领者和即兴团队成员的微信小程序。它帮助用户在排练现场快速找到合适的即兴游戏，低成本记录灵感和反馈，并把零散经验逐步沉淀为可复用的方法卡、游戏变体和团队复盘资产。

产品核心不是单纯的“即兴游戏百科”，而是临场可用的“找游戏 + 快记录 + 可沉淀”工具。

## 2. 核心用户

- 即兴主理人：需要组织排练、选择游戏、记录反馈、沉淀带领经验。
- 排练带领者：需要在现场快速查游戏、加游戏、暂停记录、结束后复盘。
- 团队成员：需要回看团队排练记录，理解练习目标、游戏反馈和复盘结论。

## 3. 核心问题

- 游戏散落在聊天、笔记和记忆里，临场找不到合适游戏。
- 排练中不方便完整记录，很多 Keep / Try 结束后就忘了。
- 灵感很多，但缺少整理机制，难以变成下次可复用的方法。
- 团队排练经验无法持续积累，每次排练都像重新开始。
- 临场操作需要足够快，避免长表单、多跳转和频繁二次确认。

## 4. 产品目标

- 帮用户快速找到“现在适合玩”的即兴游戏。
- 帮用户用最低成本记录灵感、游戏实践和排练过程。
- 帮用户把灵感、反馈和复盘转成可复用的方法卡。
- 帮团队保留排练记录，形成长期可回看的练习资产。

## 5. 核心数据对象

### 5.1 游戏

游戏是即兴游戏条目，包含名称、主分类、描述、人数、时长、标签、适用判断、玩法步骤、带领提示、变体、翻车点和相似游戏。

游戏支持搜索、标签筛选、状态筛选、列表/卡片视图、收藏、标记玩过、快速添加自定义游戏、进入详情和开始游戏。快速添加只要求名称和主分类，标签、描述、玩法步骤可保存后继续完善。

### 5.2 灵感

灵感是原始输入，强调快速留下来。它可以是粗糙的、不完整的、只有一句话的想法。

灵感支持文字输入、语音速记、关联游戏、关联排练、保存为灵感，以及进一步沉淀为方法卡。

### 5.3 排练

排练是一次团队或个人练习，包含团队、时长、目标、游戏计划、单个游戏反馈和最终整体复盘。

排练支持快速开启、添加游戏、切换游戏状态、记录 Keep / Try、暂停、继续、结束并复盘。

### 5.4 游戏实践反馈

游戏实践反馈是“某一次玩某个游戏”的记录，不等同于整场排练。它可以单独保存，也可以加入当前排练或新建排练。

核心字段包括当前游戏、人数、关联排练/场景、效果反馈、Keep、Try、下次提醒。

### 5.5 方法卡

方法卡是整理后的个人沉淀，强调下次可复用。它可以来自灵感、游戏实践反馈或排练复盘。

方法卡可包含带领提醒、游戏变体、台词想法、排练复盘和下次计划。我的页集中展示待整理来源，用户可手动将高价值记录沉淀为方法卡，并按来源筛选个人沉淀。

## 6. 当前页面结构

当前小程序保留 3 个主 Tab：

- 发现：找游戏、筛选、随机抽卡、添加游戏、进入游戏详情。
- 记录：语音速记、记录灵感、快速开启排练、查看今日记录。
- 我的：查看待整理、个人沉淀、灵感记录、游戏记录和团队排练记录。

当前已声明子页面：

- 游戏详情页：`pages/game-detail/index`
- 灵感记录页：`pages/inspiration-edit/index`
- 游戏实践反馈页：`pages/game-feedback/index`
- 排练记录页：`pages/rehearsal-record/index`
- 排练复盘页：`pages/rehearsal-review/index`
- 团队排练记录页：`pages/team-records/index`
- 游戏记录页：`pages/game-records/index`

当前容器分工：

- 轻量半弹窗：筛选、抽卡、关联选择、语音速记。
- 任务型半弹窗：快速添加游戏、快速开启排练，只承载一个临场任务。
- 子页面：完整编辑、深读、长表单、排练过程和复盘。

当前反馈规则：

- 普通轻提示使用柔和白色半透明胶囊。
- 待同步提示使用暖色样式，但仍不阻塞当前任务。

## 7. 核心用户链路

### 7.1 找游戏并开始游戏

发现页搜索或筛选游戏 -> 进入游戏详情 -> 点击开始游戏 -> 结束并复盘 -> 游戏实践反馈页 -> 填写反馈 -> 保存反馈 -> 可继续沉淀为方法卡。

### 7.2 随机抽卡并开始游戏

发现页点击随机浮球 -> 条件抽卡半弹窗 -> 再抽一张或查看详情 -> 进入游戏详情 -> 开始游戏 -> 结束并复盘 -> 保存记录。

### 7.3 快速记录灵感

记录页点击大麦克风 -> 语音速记半弹窗 -> 归类为灵感 -> 保存为灵感或进入灵感记录页整理。

### 7.4 开启排练

记录页点击记录排练 -> 快速开启排练半弹窗 -> 选择团队、时长、目标和游戏添加方式 -> 进入排练记录页。

### 7.5 排练中记录

排练记录页 -> 查看游戏计划 -> 切换未开始/进行中/已完成 -> 记录每个游戏的 Keep / Try -> 可暂停或继续添加游戏。

### 7.6 结束并复盘

排练记录页点击结束并复盘 -> 排练复盘页 -> 填写整体 Keep / Try / 下次提醒 -> 保存复盘 -> 可沉淀方法卡。

### 7.7 回看资产

我的页 -> 待整理 -> 查看详情 -> 手动沉淀方法卡；我的页 -> 个人沉淀/灵感记录列表 -> 查看详情卡片；我的页 -> 团队排练记录 -> 查看排练记录。

## 8. MVP 必须具备

- 游戏库浏览、搜索、筛选、收藏、玩过状态。
- 游戏详情和添加自定义游戏。
- 快速记录灵感。
- 游戏实践反馈。
- 快速开启排练、排练游戏计划、游戏状态和 Keep / Try。
- 排练复盘。
- 我的页资产回看。
- CloudBase 数据隔离、统一 action、基础错误提示。

## 9. 可以后置

- 团队多人协作编辑。
- 复杂推荐算法。
- 方法卡公开分享。
- 高级统计报表。
- 自动生成完整排练模板。
- 复杂语音实时转写和 AI 摘要。

## 10. 待确认问题

- 个人沉淀是否只对自己可见，还是可分享给团队。
- 团队排练记录是否需要多人共同编辑。
- 语音记录是实时转写，还是录完后生成摘要。
- 游戏库以官方种子库为主，还是用户自建为主。
- 排练中的游戏状态是否需要“跳过”“临时加入”“待复盘”等更细状态。
- 方法卡是否需要支持从多个记录合并生成。
- 数据是否需要区分个人空间和团队空间。

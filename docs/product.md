# 即兴工具箱产品文档

版本：MVP 文档版
更新时间：2026-06-21
当前实现：`wechat-cloudbase-app/`

## 1. 产品定位

即兴工具箱是一款面向即兴主理人、排练带领者和即兴演员的微信小程序。它帮助用户管理个人私密训练资产：快速查素材、记录灵感、开始练习、编排排练，并把零散经验沉淀为可复用的方法卡和复盘资产。

产品核心不是单纯的“即兴游戏百科”，而是临场可用的“素材发现 + 快速记录 + 练习复盘 + 个人沉淀”工具。

## 2. 核心用户

- 即兴主理人：需要备课、选素材、带练、控场、复盘和沉淀带领经验。
- 即兴演员：需要练习角色、身体、声音、叙事、关系、才艺模仿和失败复原。
- 排练带领者：需要在现场快速查素材、加素材、暂停记录、结束后复盘。

## 3. 核心问题

- 素材散落在聊天、课程、笔记、演出记忆和网页里，临场找不到。
- 即兴训练不只需要游戏，还需要角色、才艺、格式、主理话术、技巧和复盘方法。
- 排练中不方便完整记录，很多 Keep / Try 结束后就忘了。
- 灵感很多，但缺少整理机制，难以变成下次可复用的方法。
- 临场操作需要足够快，避免长表单、多跳转和频繁二次确认。

## 4. 产品目标

- 帮用户快速找到“现在适合用”的即兴素材。
- 帮用户用最低成本记录灵感、素材练习和排练过程。
- 帮用户把灵感、练习复盘和排练复盘转成可复用的方法卡。
- 帮用户保留私密训练记录，形成长期可回看的个人训练资产。

## 5. 核心数据对象

### 5.1 素材 Material

素材是 App 内所有可查阅、可练习或可参考内容的上位对象。

素材类型包括：

#### 游戏类素材
- **定义**：热身、短篇、限制、关系、叙事、猜词、身体、音乐等即兴游戏。
- **用途**：临场可用、快速启动、明确规则、适合新手入门和演出暖场。
- **数量**：约 100+ 个游戏（来源：improv_materials_games.json）
- **示例**：名字接龙变奏、Zip Zap Zop、一词一句故事、礼物传递等。

#### 角色（Character）
- **定义**：职业、身份、关系、状态、经典类型人物、非人角色等。
- **用途**：快速建立角色、场景关系、身份确认、人物塑造。
- **数量**：2+ 个素材
- **示例**：30秒建立角色、身体语言与声音、展示而非讲述等。

#### 才艺（Talent）
- **定义**：舞蹈、歌曲、口音、模仿、运动、职业技能、仪式动作等。
- **用途**：才艺模仿、识别点建立、观众快速认出风格。
- **数量**：1+ 个素材
- **示例**：三拍识别舞步、方言口音模仿、职业技能展示等。

#### 格式（Format）
- **定义**：短篇结构、长篇格式、开场、转场、回收、蒙太奇、叙事结构等。
- **用途**：演出结构、场景编排、叙事框架、长即兴格式。
- **数量**：7+ 个素材
- **示例**：Harold格式、短篇开场、场景转场、蒙太奇回收、Theatresports等。

#### 主理（Hosting）
- **定义**：主持词、规则讲法、带练提示、控场话术、破冰话术、复盘问题等。
- **用途**：主理人备课、控场、带练、复盘引导。
- **数量**：5+ 个素材
- **示例**：排练开场主持词、即兴演出提示词全指南、MC主持技巧、演出准备、拼盘演出策划等。

#### 技巧（Technique）
- **定义**：Yes And、接纳、聆听、平台建立、关系推进、身体空间、叙事构建等。
- **用途**：核心原则训练、基础能力培养、进阶技巧掌握。
- **数量**：20+ 个素材（来源：improv_materials_articles.json）
- **示例**：自发性训练、Yes And原则、积极聆听、拥抱失败、地位理论、CROW框架、情绪表达等。

#### 复盘（Review）
- **定义**：Keep、Try、下次提醒、观众反馈、团队观察、失败案例、成功模式等。
- **用途**：练习后复盘、排练总结、经验沉淀、下次提醒。
- **数量**：1+ 个素材
- **示例**：Keep/Try复盘卡、KISS复盘方法、What/So What/Now What等。

#### 路径（Path）
- **定义**：演员学习路径地图、完整训练路径等低频参考型素材。
- **用途**：训练认知地图、学习进度参考、路径规划。
- **数量**：5+ 个素材
- **示例**：演员学习路径地图、入门阶段学习路径、进阶阶段学习路径、高阶阶段学习路径等。
- **特殊**：只支持查看、收藏和筛选跳转，不进入训练计时、抽卡训练池或排练计划。

素材支持搜索、筛选、分类浏览、收藏、标记练过、最近使用、随机抽卡和详情查看。除 `路径` 外，其他素材支持开始训练、暂停、结束复盘。`路径` 只支持查看、收藏和筛选跳转。

**素材总计**：约 50+ 素材（不含游戏库），游戏库约 100+ 个游戏。

### 5.2 灵感

灵感是原始输入，强调快速留下来。它可以是粗糙的、不完整的、只有一句话的想法。

灵感支持文本输入、关联素材、关联排练、保存为灵感，以及进一步沉淀为方法卡。我的页灵感记录支持按整理状态筛选：全部、待整理、训练线索、排练线索、已沉淀。训练线索和排练线索只是当前版本的本地用途标记，不会创建练习、创建排练或回写历史排练。当前版本不提供 app 内置音频采集、转写或自动摘要；需要口述输入时，使用系统输入法自带的语音转文字能力。

### 5.3 排练

当前个人版中的“排练”是一次个人多素材练习 session，包含名称、时长、目标、素材计划、单个素材 Keep / Try 和最终整体复盘。`teamName` / 排练对象只作为历史记录字段，不代表多人协作空间、成员权限或共同编辑能力。

排练支持快速开启、添加素材、切换素材状态、记录 Keep / Try、暂停、继续、结束并复盘。排练计划可加入除 `路径` 外的多类型素材。

### 5.4 练习复盘

练习复盘是“某一次练某张素材”的记录，不等同于整场排练。它可以单独保存，也可以关联当前排练。

核心字段包括当前素材、评分、复盘文本、人数、关联排练/场景、下次提醒和可选附件。当前体验升级方向是把首屏收敛为 `1-10` 分评分和“本次复盘”，人数、提醒和关联 session 放入更多信息。

练习记录是单素材维度的个人训练资产。素材详情页应展示该素材的练习记录摘要，例如平均分、训练次数、最近记录和 `查看全部` 入口；完整筛选和附件回看进入练习记录页承接。

### 5.5 方法卡

方法卡是整理后的个人沉淀，强调下次可复用。它可以来自灵感、素材练习复盘或排练复盘。

方法卡可包含带领提醒、素材变体、台词想法、排练复盘和下次计划。我的页集中展示待整理来源，用户可手动将高价值记录沉淀为方法卡；灵感类待整理项也可标记为训练线索或排练线索，并在灵感记录中筛选回看。

## 6. 当前页面结构

当前小程序保留 3 个主 Tab：

- 发现：素材发现、搜索筛选、随机抽卡、添加素材、进入素材详情。
- 记录：文本记录灵感、记录练习、快速开启排练、查看暂停任务。
- 我的：待整理、个人沉淀、灵感记录、练习记录和排练记录。当前个人版不表达团队协作能力。

当前已声明子页面：

- 素材详情页：`pages/material-detail/index`
- 灵感记录页：`pages/inspiration-edit/index`
- 练习复盘页：`pages/practice-feedback/index`
- 排练记录页：`pages/rehearsal-record/index`
- 排练复盘页：`pages/rehearsal-review/index`
- 排练记录回看页：`pages/team-records/index`（历史路径名保留，产品口径按个人多素材 session 解释）
- 练习记录页：`pages/practice-records/index`
- 设置页：`pages/settings/index`
- 使用帮助页：`pages/help/index`
- 隐私政策页：`pages/privacy/index`

说明：页面路径已统一使用素材与练习语义，不再保留旧游戏路由口径。

## 7. 核心用户链路

### 7.1 找素材并开始练习

发现页搜索或筛选素材 -> 进入素材详情 -> 点击开始训练 -> 暂停或结束复盘 -> 练习复盘页 -> 填写反馈 -> 保存记录 -> 可继续沉淀为方法卡。

### 7.2 随机抽卡并开始练习

发现页点击随机浮球 -> 从非路径素材中抽一张 -> 进入素材详情 -> 开始训练 -> 结束复盘 -> 保存记录。

### 7.3 查看路径参考

发现页进入分类模式 -> 点击 `学习地图` 或 `训练路径` -> 半弹窗查看完整路径 -> 可编辑并保存为我的路径副本 -> 按能力找素材或回到素材列表。

### 7.4 快速记录灵感

记录页直接填写灵感文本 -> 保存为灵感 -> 可进入灵感记录页整理。

### 7.5 开启排练

记录页点击记录排练 -> 快速开启排练半弹窗 -> 选择团队、时长、目标和素材添加方式 -> 进入排练记录页。

### 7.6 排练中记录

排练记录页 -> 查看素材计划 -> 切换未开始/进行中/已完成 -> 记录每个素材的 Keep / Try -> 可暂停或继续添加素材。

### 7.7 结束并复盘

排练记录页点击结束并复盘 -> 排练复盘页 -> 填写整体 Keep / Try / 下次提醒 -> 保存复盘 -> 可沉淀方法卡。

### 7.8 回看资产

我的页 -> 待整理 -> 查看详情 -> 手动沉淀方法卡；我的页 -> 个人沉淀/灵感记录/练习记录/排练记录 -> 查看详情。

### 7.9 查看帮助与提交反馈

我的页右上角 -> 设置 -> 使用帮助或意见反馈。使用帮助为随版本发布的静态内容；意见反馈在任务型半弹窗中填写，服务端确认写入后才提示成功。

## 8. MVP 必须具备

- 素材库浏览、搜索、筛选、收藏、练过状态。
- 8 个素材类型完整展示：游戏、角色、才艺、格式、主理、技巧、复盘、路径。
- 素材详情和添加自定义素材。
- 非路径素材训练、暂停、结束复盘。
- 路径素材查看、收藏、筛选跳转；学习地图和训练路径可保存为我的路径副本，但不进入训练计时。
- 快速记录灵感。
- 练习复盘。
- 快速开启排练、排练素材计划、素材状态和 Keep / Try。
- 排练复盘。
- 我的页资产回看。
- CloudBase 数据隔离、统一 action、基础错误提示。
- 设置、静态使用帮助、应用内意见反馈、隐私政策和账号注销入口。

## 9. 当前版本边界

当前 MVP 版本聚焦个人私密训练空间，不引入公开广场、关注、公开素材库或复杂团队权限。

- 素材库以用户自建和手动导入为主；官方或示例素材库只作为初始化和参考材料。
- 当前版本不提供 app 内置音频采集、转写、摘要或 AI 整理；口述输入交给系统输入法的语音转文字能力承接。
- 当前版本不提供纯手势“上滑/下滑进入新页面”；跨页面导航必须有可见入口。
- 排练中的素材状态保持现有简化状态，不增加“跳过”“临时加入”“待复盘”等更细状态。
- 数据暂不区分个人空间和团队空间；团队名称、排练对象等只作为记录字段存在，不产生团队空间、成员或权限。
- 方法卡暂不支持从多个记录合并生成；当前只支持用户从单条灵感、练习复盘或排练复盘中手动沉淀。
- 团队协作编辑、团队空间、多角色权限、团队统计、方法卡分享和公开传播能力不属于当前版本上线范围。

## 10. 后续版本方向

- 团队多人协作编辑。
- 方法卡分享和公开传播。
- 多空间、多角色和团队权限。
- 从多个记录合并沉淀方法卡。
- 复杂推荐算法。
- 高级统计报表。
- 自动生成完整排练模板。

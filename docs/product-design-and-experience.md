## Part 1. 新用户或无数据状态页面的显示和逻辑

### 1. 背景与目标

当前项目已经逐步去掉业务 mock 数据，并明确当前开发阶段不启用 `store` 本地持久化缓存。用户第一次打开小程序、数据库还未导入、或某类数据本来就为空时，页面会更频繁进入真实的无数据状态。

这要求页面不再使用“留白”或“默认卡片”掩盖空数据，而要清楚地告诉用户：

- 当前为什么没有内容。
- 下一步最应该做什么。
- 哪些二级功能此时还不适合先用。

本部分当前优先解决发现页和条件抽卡，因为这两处已经出现最明显的“空白页面 / 假装有内容 / 条件不一致”问题。

### 2. 适用范围

本部分当前覆盖：

- `pages/discover/index`
- 发现页 `全部 / 分类` 任务切换
- 发现页完整筛选弹层
- 发现页“添加素材”入口与创建表单
- 发现页“条件抽卡”浮球和抽卡弹层

本部分后续扩展：

- `pages/record/index`
- `pages/mine/index`
- `pages/team-records/index`

### 3. 状态定义

#### 3.1 首次打开无素材库

定义：

- 当前账号首次进入发现页。
- 素材集合为空，或当前账号可见的素材列表为空。
- 页面还没有任何可以展示、搜索、筛选、抽卡的素材。

目标：

- 让用户立刻理解“现在没有素材库”。
- 主动作聚焦在“添加第一个素材”。

#### 3.2 有素材库，但当前筛选无结果

定义：

- 素材库本身不为空。
- 当前筛选、搜索词或条件抽卡条件下，没有匹配的结果。

目标：

- 告诉用户“不是没有游戏，而是当前条件太窄”。
- 给出放宽条件或清空筛选的直接动作。

#### 3.3 云端失败

定义：

- 请求失败、权限不足或网络异常，导致当前数据无法拉取。

目标：

- 清楚提示“当前取数失败”。
- 提供重试或返回上一步的动作。

说明：

- 当前阶段不启用本地持久化缓存，因此失败时不应伪装成旧数据仍然可用。
- 写入失败时保留当前表单内容，不关闭页面，并提供明确重试；不将未入库对象加入历史列表。

### 4. 页面显示策略总则

#### 4.1 总展示优先级

页面展示优先级统一为：

1. 云端成功结果
2. 空态
3. 错误态

规则：

- 云端成功返回空数组时，进入空态。
- 云端失败时，进入错误态。
- 不允许再回退旧本地持久化缓存。

#### 4.2 空态不是留白

空态必须包含：

- 明确标题
- 一句解释
- 一个主动作

可选包含：

- 次动作
- 示例提示
- 当前功能暂不可用的说明

不建议：

- 只留空白区域
- 继续显示和当前状态不一致的默认卡片
- 在没有数据时仍保留大量不可用的二级入口

### 5. 发现页：首次打开无素材库

#### 5.1 页面目标

当用户第一次打开发现页且没有任何素材数据时，页面的唯一主任务是帮助用户开始建立第一个最小可用素材库。

#### 5.2 页面显示规则

当 `materials.length === 0` 时：

- 发现页只显示一个大卡片，不再保留完整发现页骨架。
- 大卡片内容固定为：
  - 主标题：`添加素材`
  - 辅助文案：`马上开玩`
  - 主按钮：`添加素材`
- 大卡片排版固定为：主标题与辅助文案在卡片中部居中显示，主按钮独立放在文案下方，不与文案并排，也不悬在卡片正中央。
- 不提供次按钮。
- 点击主按钮后，直接打开当前已有的“添加素材”快速收录底部弹层。

#### 5.3 页面层级调整

无素材库时：

- 隐藏 `全部 / 分类` 切换。
- 隐藏搜索框。
- 隐藏分类网格。
- 隐藏 `全部素材` 标题区。
- 隐藏 `筛选` 按钮。
- 隐藏素材列表。
- 隐藏右下角抽卡浮球。

原因：

- 此时筛选和抽卡不是首要动作。
- 用户此时最需要的是明确的起步动作，而不是复杂功能。

#### 5.4 推荐文案基准

- 主标题：`添加素材`
- 辅助文案：`马上开玩`
- 主动作：`添加素材`

#### 5.5 状态切换规则

- 大卡片只用于“当前没有任何素材”的状态。
- 一旦当前会话或云端结果中存在至少一个素材，发现页立即恢复到正常有数据状态。
- 恢复后重新显示：
  - `全部 / 分类` 任务切换，默认选中 `全部`
  - 全部模式搜索框
  - `全部素材` 标题区与 `新建 / 筛选`
  - 全部素材列表
  - 条件抽卡入口
- 添加成功后，底部弹层关闭，发现页直接显示新加入的素材。

### 6. 发现页：全部模式、分类模式与完整筛选

#### 6.1 默认全部模式

发现页有素材时默认进入 `全部` 模式，而不是分类模式。

全部模式负责临场速查和素材管理，应展示：

- 搜索框
- `新建`
- `筛选`
- 全部素材列表
- 条件抽卡入口

不再保留 `列表 / 卡片` 这类纯展示样式切换。发现页的切换只表达用户任务：`全部` 和 `分类`。

#### 6.2 分类模式

分类模式用于浏览素材体系，不承担高意图搜索。

- 分类模式不展示搜索框。
- 分类总览不展示 `新建` 按钮。
- 主体先展示 `素材类别` 主导航区，再展示 `训练系统参考` 次级区。
- 主导航区展示当前可见素材中数量大于 0 的固定类别卡片。
- 类别卡片展示完整类别名、服务端全库数量和一行用途提示；数量不使用当前分页结果计算。
- 类别卡片保持轻量目录样式，避免大面积高饱和色块和过重阴影。
- 点击类别卡进入类别详情态。
- 次级区下方提供两个轻量入口：`学习地图` 和 `训练路径`。
- 两个入口默认只展示标题、1 行摘要、关键阶段标签和 `查看` 动作，不在当前页点击展开。
- 点击入口后打开底部半弹窗展示完整步骤流。
- `学习地图` 半弹窗显示：入门基础、身体与声音、角色关系、叙事结构、格式与主理。
- `训练路径` 半弹窗显示：个人热身、双人场景、小组排练、演出准备、演后复盘。
- 两个入口支持编辑我的版本：官方预设保持只读，用户首次编辑时保存为自己的 `路径` 素材副本；后续编辑更新该副本。
- 个人路径副本使用 `relatedMaterialId` 关联预设 key，仍保持 `referenceOnly: true`，不进入抽卡训练池、训练计时或排练计划候选。
- 分类卡中的 `路径` 表示“参考型路线素材类别”；下方 `训练路径` 表示“训练系统参考路线”，文案和样式需要明确区分。

类别详情态用于在某一类素材中继续查找：

- 顶部显示纯箭头返回入口和独立当前类别名，不把箭头与类别名合并成同一个文字按钮。
- 搜索框 placeholder 使用 `在游戏里搜索` 这类类别内文案。
- 列表只展示当前类别内素材。
- 筛选继续可用，但素材类型由当前类别锁定，不重复展示。
- 当前类别本身没有素材时，空态提供“添加当前类别素材”单一主动作，并在现有添加素材表单中预选该类型。
- 当前类别或全部素材因搜索、能力、场景、状态条件而无匹配结果时，空态只说明结果，不提供“清空条件”；用户通过搜索框、筛选弹层或返回入口自行调整。

#### 6.3 完整筛选放入弹层

完整筛选继续放在底部筛选弹层，承载：

- 素材类型
- 训练能力
- 场景
- 状态

筛选使用动态分面统计：类型、训练能力、使用场景和状态都只展示当前范围内仍可产生结果的未选条件；`全部` 始终保留，已选择条件即使数量变为 0 也继续显示，确保用户可以取消。计算某个维度的数量时应用搜索词和其他维度条件，但忽略该维度当前条件；所有数量均由服务端按完整可见素材集合统计，不受前端分页影响。

#### 6.4 无数据时的筛选策略

当没有素材库时：

- `全部 / 分类` 切换不应承担主视觉。
- 完整筛选按钮不应作为主动作。
- 筛选本身可以隐藏，或展示为不可用状态。

### 7. 新建游戏：哪些条件是必填，哪些不是

#### 7.1 字段分层

新建游戏时，不应把所有筛选条件都变成必填字段，也不应把完整游戏百科编辑塞进半弹窗。

应拆成三类：

- 必填内容字段
  - 素材名称 `title`
  - 至少一个结构化分类标签 `tags`
- 推荐填写字段
  - 适合人数
  - 时长
  - 标签
  - 简短描述
- 可选补充字段
  - 玩法步骤
  - 带领提示
  - 变体
- 不属于创建必填的字段
  - `saved`
  - `played`
  - `status`
  - `最近排练`

#### 7.2 分类必须按三维度结构化

分类使用结构化选择，并按 `类型`、`能力`、`场景` 三个维度分行展示。三者职责不同：

显示规则：

- `类型` 是素材上位类别，默认展示游戏、角色、才艺、格式、主理、技巧、复盘、路径，单选。
- `能力` 是训练能力维度，例如自发性、Yes And、积极聆听、角色塑造等，可多选。
- `场景` 是使用场景维度，例如临场速查、备课、排练、演出，可多选。
- 三个维度各占一行，使用横向轻量 chips，不混排成一组。

数据规则：

- `类型` 写入 `type`。
- `能力` 写入 `abilities`。
- `场景` 写入 `scenes`。
- `tags` 可保留当前选中分类的去重结果，用于兼容搜索、卡片展示和旧链路。
- 如果未选择类型，保存时使用 `游戏` 兜底。

#### 7.3 半弹窗职责

添加素材半弹窗只承载快速收录：

- 默认显示名称、类型、能力、场景、人数、时长、简短描述。
- 顶部不再显示大段说明文字，避免首屏被提示文案占用。
- 玩法步骤默认收起，作为补充项。
- 分类区域必须分成三行：`类型`、`能力`、`场景`。类型单选，能力和场景可多选。
- `适合人数` 与 `时长` 保持同一行双列布局，标签和输入框顶部对齐。
- 玩法步骤展开后，任务型半弹窗内容区继续保持独立滚动，必须能看到完整输入区和提交按钮。
- 名称、简短描述、补充玩法步骤统一使用文本输入，不提供 app 内语音按钮。
- 单行输入和多行 textarea 都优先保留完整编辑空间、文本可读性和滚动区域。
- 任务型弹层的提交、保存、开始和确认按钮统一放在底部操作区；按钮组保持居中，不拉满整行，也不允许按钮文字换行。
- 完整编辑、长步骤、带领提示和变体放到保存后继续完善或当前素材详情页内编辑。
- 发现页“添加素材”和素材详情页“编辑素材”应共享同一套字段分组、类型/能力/场景三行分类和文本输入布局，避免录入页与编辑页发生视觉或字段语义漂移。
- 发现页加载失败、空素材库、筛选无结果、随机抽卡无结果等典型空态，应统一复用同一套空态组件，只调整文案和动作，不再保留页面内平行手写结构。
- 发现页筛选面板、记录页快速开启排练、灵感页整理方向等 chips 选择区，应共享同一套筛选分组壳层，统一标题、间距、横向滚动和 wrap 行为。
- 素材录入、素材编辑、灵感编辑和复盘页统一使用文本输入；口述输入交给系统输入法自带的语音转文字能力。

这样可以保证：

- 发现页筛选、分类详情和创建表单口径一致
- 后续搜索和筛选质量更稳定

#### 7.4 轻提示

轻提示统一采用全局 toast：

- 普通成功提示为柔和白色半透明胶囊，不遮挡半弹窗表单内容。
- 非阻断注意提示使用暖色样式；保存失败使用 error 语义。
- 不使用黑色系统 toast 作为常规反馈；系统 toast 只作为拿不到当前页面实例时的兜底。
- 不再使用居中白色提示卡片作为普通反馈，避免与输入框和弹层白底重复。

#### 7.5 人数与时长的定位

人数和时长建议填写，但不强制。

如果用户首次录入时只知道名字和玩法大意，也应允许先收录，再逐步补全。

显示规则建议：

- 未填写时，页面可显示 `待补充`
- 在精确筛选和条件抽卡中，这类数据不完整的游戏权重较低

#### 7.5 标签的定位

标签用于补充固定类型、训练能力和使用场景无法表达的特征，并提升搜索和抽卡质量，但不参与一级分类管理。

建议：

- 首屏依次展示类型、能力、场景和标签；标签区域默认只显示已选标签及轻量添加入口。
- 不增加复杂标签管理器。
- 标签允许自由添加和删除，但不能替代必选的固定类型。

#### 7.6 素材详情编辑与删除

素材详情页保留查看态和编辑态切换，不新增独立编辑页。

- 只有当前用户创建的素材或当前用户的路径副本显示 `编辑`。
- 系统公共素材保持只读，不显示编辑和删除入口；未来如需改造，应走“复制为我的版本”，不直接修改公共素材。
- 编辑态复用添加素材表单的字段分组：名称、类型、能力、场景、标签、人数、时长、描述为主区，训练步骤、带领提示、变体和可能翻车点作为折叠补充项。
- 编辑态底部固定显示 `删除素材` 和 `保存修改`。
- 保存和删除必须等待云端 `material.update` / `material.delete` 成功后再更新详情页和素材列表；失败时停留当前状态并提示原因。
- 删除使用软删除，从当前用户素材库移除，不做物理删除。
- `路径` 系统素材只支持查看、收藏和筛选跳转；用户路径副本可编辑，但仍保持 `referenceOnly: true`，不进入训练计时、抽卡训练池或排练计划候选。

### 8. 条件抽卡：无数据与无匹配的显示逻辑

#### 8.1 抽卡弹层三态

条件抽卡必须明确支持三态：

1. 可抽卡
2. 无素材库
3. 当前条件无匹配

#### 8.2 无素材库时

条件：

- `materials.length === 0`

显示：

- 标题：`还没有可抽的素材`
- 说明：`先添加几个常用素材，再来条件抽卡。`
- 主按钮：`添加素材`

不应显示：

- 空白卡片
- `currentRandomMaterial.title`
- `再抽一张`
- `查看详情`

#### 8.3 当前条件无匹配时

条件：

- `materials.length > 0`
- 当前抽卡条件或发现页筛选条件下没有任何匹配结果

显示：

- 标题：`当前条件下没有匹配素材`
- 说明：`放宽条件，或者先从全部素材里抽一张。`
- 主按钮：`清空条件`
- 次按钮：`从全部素材抽`

#### 8.4 可抽卡时

条件：

- `currentRandomMaterial` 存在

显示：

- 当前抽中的素材卡片
- `查看详情`
- `再抽一张`
- 进入详情后再开始训练

标签展示规则：

- 抽卡卡片顶部按 `type -> abilities -> tags` 的顺序展示只读标签，并在展示前去重。
- 只读标签复用全局 `pill` 规格，不使用面向筛选操作的交互型 `chip` 规格。
- `tags` 只保存自定义标签，不重复保存已经结构化进入 `type`、`abilities`、`scenes` 的值。

规则：

- 只有 `currentRandomMaterial` 存在时才渲染卡片。
- 不允许在空数据场景下直接访问 `currentRandomMaterial.title` 或 `currentRandomMaterial.desc`。

#### 8.5 抽卡条件文案

抽卡头部条件必须来自真实条件，而不是写死展示。

当前建议：

- 如果用户已经在发现页设置了筛选条件，则抽卡弹层显示这些条件摘要
- 如果当前没有条件，则显示：
  - `全部素材`
  - `随机抽卡`

不建议继续写死：

- `5 分钟内`
- `新手友好`
- `高能量`

除非这些条件已经和真实数据、真实筛选逻辑完全打通。

### 9. 当前已确认决策

- 首次打开且没有素材数据时，发现页主动作是“添加第一个素材”。
- 发现页无数据时，页面只显示一个大卡片，不再保留搜索、筛选、抽卡等依赖已有数据的入口。
- 大卡片文案收敛为：`添加素材` + `马上开玩`。
- 大卡片中的 `添加素材` 按钮点击后，继续使用当前已有的底部弹层。
- 有素材时发现页默认进入 `全部` 模式，保留搜索、新建、筛选和素材列表。
- `分类` 是浏览素材体系的第二模式，不显示搜索框；点击类别后进入类别详情态，再显示类别内搜索和列表。
- 不再保留 `列表 / 卡片` 这种纯展示样式切换，也不再用横向类别 chips 承担主分类入口。
- 新建游戏时，不应把所有筛选条件都做成必填项。
- 素材名称和至少一个分类标签是最小可用核心字段。
- 分类在添加素材表单中拆为类型、能力、场景三行 chips，不合并为单组混排。
- `status`、`saved`、`played`、`最近排练` 不属于素材创建表单必填字段。
- 条件抽卡必须区分：
  - 无素材库
  - 当前条件无匹配
  - 有数据可抽
- 抽卡弹层不能再默认假设一定存在 `currentRandomMaterial`。

### 11. 后续扩展占位

本部分补充记录页、我的页、灵感编辑页和排练记录页的显示规则，作为“有数据才显示正式资产卡、无数据不强行占位”的正式约定。

#### 11.1 记录页

- 记录页首屏主任务固定为：`记录灵感`、`记录练习`、`记录排练`。
- `今天已记录` 统计行只在今日灵感或今日排练任一计数大于 `0` 时显示。
- 当今日两类记录都为 `0` 时，不保留可点击统计入口，也不使用 Toast 催促点击空记录。
- `今日推荐` 允许保留两种价值：
  - 有素材库时，推荐一个真实游戏进入详情。
  - 没有素材库时，提醒用户可以先记录灵感，作为首屏的轻引导卡。
- `今日推荐` 不应用于堆叠泛化说明文案；它只承担“推荐一个动作”的职责。
- 今日记录弹层在空数据时必须显示页面内空态说明，不应出现空白弹层。

#### 11.2 我的页

- `我的` 页继续保留“全空首登态”。
- 一旦进入非全空状态，资产区不再默认展示所有零数据入口卡。
- 只有真正存在数据的资产，才显示为正式资产卡：
  - 方法卡
  - 灵感记录
  - 练习记录
  - 排练记录
- `待整理` 作为高价值下一步动作，保持按数量条件显示。
- 当已发生的资产仍较少时，可补一张轻引导卡，统一提示“继续记录”，而不是摆出多张 `0 条` 资产卡。
- 统计卡不再因为页面需要平衡而强行显示；只有进入真实积累阶段后，才作为资产概览出现。

#### 11.3 灵感编辑页

- 灵感编辑页仍以“先保存灵感”为主，不强制用户先补全关联。
- 当没有可关联的游戏或排练，且当前也没有既有绑定值时，可整体隐藏“可选关联”模块，缩短表单。
- 关联弹层在无可选项时必须显示空态说明和退出动作，不允许只剩空白列表。

#### 11.4 排练记录页

- 排练记录页属于过程态页面，没有计划游戏时也不能只剩空容器。
- `素材计划` 在 `planMaterials.length === 0` 时，应显示单一 CTA 空态，明确下一步是“加素材”。
- `添加到排练` 弹层在无可选游戏或搜索无结果时，应显示空态说明：
  - 没有素材库时，引导去发现页添加。
  - 搜索无结果时，引导清空关键词继续浏览。
- `调整排练计划` 弹层在没有任何计划项时，同样应显示空态说明，而不是只保留标题和保存按钮。

#### 11.5 回看页列表

- `practice-records` 和 `team-records` 属于历史回看页，不应在空数据时继续显示 `0 条` 摘要卡。
- 当列表为空时，主视觉应直接切到空态卡，说明这里未来会回看什么，并提供返回主链路的动作。
- 当列表有内容时，再恢复顶部摘要卡与列表结构。
- 请求失败但当前会话仍有记录时，可以继续展示当前会话记录，同时通过 notice 说明云端暂不可用。

### 12. 首次进入与我的页首登展示

#### 12.1 登录策略

- 从产品定位上看，小程序首次打开不应强制登录。
- 当前推荐采用轻登录：
  - `发现` 和 `记录` 可先直接使用。
  - `我的` 页允许进入，但首屏以价值说明和空态引导为主。
  - 涉及保存、同步、个人沉淀、团队记录等归属型动作时，再逐步建立身份感知。

原因：

- 产品核心是临场工具，不是先做账号体系。
- 首次使用时应优先让用户快速理解“这个工具能帮我什么”，而不是先完成身份步骤。

#### 12.2 我的页首登目标

- 当用户第一次进入 `我的` 页且还没有任何个人数据时，页面不应伪装成成熟个人主页。
- 这一状态的目标是：
  - 告诉用户这里未来会沉淀什么
  - 告诉用户从哪里开始积累
  - 降低“还没有内容”的挫败感

#### 12.3 我的页首登显示规则

- 我的页首登页采用：`三段式布局`。
- 三段职责固定为：
  - 头部价值卡：说明这里是什么
  - 状态统计卡：说明现在是什么状态
  - 底部填充行动卡：说明接下来做什么
- 顶部主卡改为价值引导卡，而不是默认资料卡。
- 顶部主卡保留个人资料入口：
  - 默认名字可显示为 `即兴主理人`
  - 支持用户后续修改名字和上传头像
  - 卡片右上角提供 `编辑` 按钮
  - 点击 `编辑` 后使用轻量半弹窗承载资料修改
  - 资料半弹窗短内容时贴合内容高度，只有超过可视高度时才进入内部滚动
  - 头部采用 `头像 + 标题信息 + 编辑按钮` 的独立三列结构
  - `编辑` 按钮属于卡片角标操作，不通过标题正文让位来实现位置
- 主卡说明文案继续表达价值引导：
  - `把热身、关系和灵感收集成一套顺手可用的排练工具，让每次现场都能更快进入状态。`
- “我的”页右上角提供独立设置入口；个人资料编辑弹窗只负责头像、名字和剧团。
- 主题切换迁移到“设置 -> 外观主题”：
  - 支持在“默认模式”与“沉浸模式（theme-vivid）”间一键切换。
  - 切换结果在当前设备即时全局生效（涵盖各主页面、子页面、TabBar 与所有半弹窗），并作为本地 UI 偏好保存在本机。
  - 主题切换不修改云端数据，也不参与账号、团队或业务记录同步。

#### 12.4 我的页首登下方结构

- 统计卡可以保留为 `0`，但应补充一句轻说明，避免像完整成绩面板。
- 底部卡属于填充行动卡，而不是普通说明卡。
- 首版可弱化原有三张入口卡，改为一张总说明卡，统一解释：
  - 方法卡会沉到这里
  - 灵感会沉到这里
  - 排练记录会沉到这里
- 个人卡片标签固定只保留 `即兴主理人` 和 `个人空间`，不再显示 `慢慢积累`、`排练带领者` 等旧标签。
- 资料弹窗里的 `上传头像` 属于局部编辑按钮，不复用全局通用 `small-btn` 视觉规格。
- 资料弹窗头像区采用 `头像预览 + 说明 + 上传按钮` 组合，不单独摆放通用胶囊按钮。
- 底部卡内部继续拆成三层：
  - 正文区
  - 标签区
  - 按钮区
- 位置关系固定为：
  - 正文区在视觉中部
  - 标签区在正文区下方
  - 按钮区贴近卡片底部
- 不建议在首登时继续显示伪默认资料，例如：
  - `上海`
  - `即兴龄 2 年`
  - `开心即兴团`

#### 12.5 何时建立身份感知

- 不强依赖身份的动作：
  - 浏览游戏
  - 搜索和筛选
  - 查看详情
- 建议在这些动作发生时再建立身份感知：
  - 添加自定义素材
  - 保存灵感
  - 保存练习复盘
  - 开启或保存排练
  - 查看个人沉淀和团队记录

规则：

- 登录或授权提示应尽量绑定到真实保存和同步动作。
- 不建议在用户第一次打开小程序时就用整页登录阻断主链路。

## Part 2. 当前已确认交互基础设施

本部分沉淀 2026-06-05 至 2026-06-06 连续整改中已经确认的实现方向，后续页面调整应沿用这些规则，不再局部反复试错。

### 1. 真机安全区布局

页面顶部不按固定 rpx 猜测，而应使用微信胶囊和系统信息计算布局变量：

- `--nav-row-top`
- `--nav-row-height`
- `--content-safe-top`
- `--page-side`
- `--page-bottom`

主页面内容从胶囊行下方开始。子页面返回箭头与胶囊同一行，标题和卡片从胶囊行下方开始。

返回按钮规则：

- 只显示箭头。
- 不显示“返回”文字。
- 与页面内容左边界对齐。

### 2. 临场表单密度

练习复盘页属于临场记录页，不应把所有字段都放在首屏。

首屏优先级：

1. 记录对象
2. `1-10` 分评分
3. 本次复盘
4. 保存按钮

Keep / Try 不再作为首屏两段输入出现，合并为“本次复盘”。评分使用 `1-10` 整数滑块，拖动时实时反馈当前分数和“需调整 / 一般 / 很有效”状态；界面只保留 `1 / 5 / 10` 关键刻度，不再铺满 10 个评分 chip。人数、关联排练/个人多素材 session 和下次提醒属于更多信息，默认折叠，用户需要时再展开。

照片和短视频作为练习记录附件处理，不进入编辑器模型。未保存附件可直接删除；已保存附件删除前必须二次确认。附件只支持查看和删除，不支持剪辑、标注、转码或独立媒体库。

记录页中的“快速开启排练”保持轻量，但目标输入要兼顾高频模板和开放表达：

- 默认提供固定推荐目标，例如 `身体到场`、`关系建立`、`叙事`、`演出前`。
- 目标应支持追加一个自定义值，并与已选固定目标一起写入排练记录。
- 目标按钮文字必须保持单行，不通过文字换行解决布局问题；当目标较多时，优先使用横向滚动。

### 3. 底部固定操作

表单页底部操作按钮可以固定，但不能形成大块纯白遮挡。当前采用轻毛玻璃/半透明操作栏，并通过内容底部预留保证最后一个输入项能滚动到操作栏上方。

如果某个任务型弹层的固定底栏造成视觉遮挡或底部空白，应优先回到内容流末尾按钮，而不是硬套固定栏。

固定底部操作栏的按钮组应保持水平居中，避免按钮被强行拉满整行；所有按钮文案保持单行显示。

### 4. 保存与沉淀

保存动作必须直接完成，不再强制弹出“记录确认”：

- 练习复盘保存后直接保存。
- 排练复盘保存后直接完成。
- 记录页文本输入可直接保存为灵感草稿。

云端写入失败时，保留当前表单草稿并明确提示“保存失败，请重试”。当前版本没有离线同步队列，因此不展示“本地暂存”或“待同步”。

我的页集中承接保存后整理：

- 新增“待整理”入口，聚合灵感、单次素材练习和已完成排练复盘。
- 待整理详情提供“不再整理”和“沉淀为方法卡”两个基础动作；不再整理只从待整理队列移除，不删除原始记录；沉淀走 `methodCard.create`，不新增后端 action。
- 灵感来源的待整理详情额外提供“训练线索”和“排练线索”两个互斥用途标签；标签默认浅色，点击后仅在当前详情中以深色选中，不立即保存、不关闭详情，也不移出待整理队列。点击“不再整理”时忽略临时选择；只有“沉淀为方法卡”成功后才保存所选用途。原灵感始终保留，并可在“灵感记录”列表顶部按全部、待整理、训练线索、排练线索、已沉淀筛选查看。
- 待整理不是通用记录编辑器：只有灵感来源提供“编辑”入口，并复用灵感编辑页更新原记录；素材练习和排练复盘属于已发生事实，在待整理中保持只读，只允许标记、不再整理或沉淀为方法卡。
- 训练线索和排练线索是本地 UI 偏好，不创建练习、不创建排练、不回写历史排练，也不新增云端字段。
- 方法卡记录保留 `sourceType`、`sourceId` 和 `sourceTitle`，用于判断来源和避免当前会话重复整理。
- 个人沉淀列表提供来源筛选，优先支持灵感、素材练习、排练复盘。
- 排练复盘本身归属于 `rehearsal` 数据链路，只更新排练记录，不额外写入 `practiceRecord`。
- 历史排练保持只读，不允许从练习复盘页或其他归档链路回写编辑。

### 5. 查看态与编辑态

团队排练历史回看不进入可编辑排练过程页。当前策略是在 `team-records` 页面内打开只读详情，展示排练摘要、计划和 Keep / Try；如果后续需要复制为新排练或承载更复杂详情，再拆独立页面。

排练过程页继续负责进行中编辑、暂停和结束复盘。

互斥规则补充：

- 如果存在进行中或暂停中的排练，不允许新增开启游戏。
- 如果存在进行中或暂停中的游戏，不允许新增开始排练。

### 6. 筛选弹层

筛选弹层按分类分行展示：

- 人数
- 时长
- 目标
- 状态

每类一行，左侧是分类标签，右侧是该分类选项。不要把所有筛选选项混排成同一个网格。

发现页筛选弹层中的选项标签统一从左侧起始线排列，并清除原生按钮自动外边距；横向与纵向间距保持一致，一行能够容纳时不提前换行，仅在选项总宽度超过容器后自然折行。

### 7. 全局轻提示

所有页面操作提示使用统一 `toast(title)`，常规状态由页面内 `root-portal` 轻提示承载。

视觉规则：

- 普通提示使用柔和白色半透明胶囊。
- 非阻断注意态使用暖色胶囊，不用于写入失败。
- 黑色系统 toast 只做无法获得当前页面实例时的兜底。
## 第二部分：信息架构与页面链路

更新时间：2026-06-19

### 2.1 设置、帮助与反馈

- “我的”页设置入口在首登空态和有数据状态下都可见，不新增底部 Tab。
- 设置页按帮助、偏好、关于、账号四组组织；主题和注销不再混入个人资料编辑。
- “我的”页设置入口使用左上角低强调文字按钮，不与个人资料主卡争夺视觉注意力。设置页卡片正文左对齐，跳转动作位于卡片右侧且保持单行。
- 外观主题使用卡片右侧双段滑动控件；点击“默认”或“沉浸”时滑块平滑切换，并立即应用和保存当前设备主题。
- 使用帮助为静态页面，覆盖快速开始、素材训练、素材分类与动态筛选、灵感、排练、方法卡、数据保存和常见问题，不提供搜索或云端内容配置。分类说明必须明确：`type` 必选且单选，`abilities` 和 `scenes` 可多选，`tags` 可自由添加但不创建一级分类。
- 意见反馈使用任务型半弹窗，禁止点遮罩误关。类型必选，正文限制 10–500 字，联系方式选填且最多 100 字。
- 提交中禁用重复操作；失败保留表单，成功后清空并关闭。意见反馈只保留在设置页，隐私政策页仅说明联系路径，不重复提供反馈按钮。

## 3. 发现页结构

发现页保留底部 Tab 名称 `发现`，内容升级为素材发现。

有素材时默认进入 `全部` 模式，首屏顺序：

1. `全部 / 分类` 任务切换，默认选中 `全部`。
2. 搜索框。
3. `全部素材` 标题区，右侧保留 `新建` 和 `筛选`。
4. 全部素材列表或筛选结果。
5. 随机抽卡悬浮按钮。悬浮球保持可拖动，拖动结束后吸附到左右安全边缘；活动范围避开顶部系统安全区与底部 TabBar，位置作为纯 UI 偏好保存在当前设备。拖动不触发抽卡，轻点才打开随机抽卡弹层；TabBar 隐藏或页面弹层打开时，悬浮球同步隐藏。视觉使用更大的品牌橙黄圆形背景、深色文字、浅色描边和柔和阴影，不使用系统 emoji。

`分类` 模式不展示搜索框，也不显示 `新建` 按钮；主体分为两层：

1. `素材类别` 主导航区：2x4 轻量宫格，负责进入各类别详情。
2. `训练系统参考` 次级区：收纳 `学习地图`、`训练路径` 两个轻量入口，负责提供训练认知地图，不与类别入口争主视觉。

点击某个类别后进入类别详情态：顶部显示纯箭头返回入口与独立类别名，搜索框变为类别内搜索，下面展示该类别的素材列表。

素材固定一级类别为：

- 游戏
- 角色
- 才艺
- 格式
- 主理
- 技巧
- 复盘
- 路径

类别入口不使用 `戏/人/艺/式/主/技/复/路` 简写。分类总览只展示当前可见素材中数量大于 0 的类别，每个入口展示完整类别名、服务端全库数量和一行用途提示；新建和编辑表单始终提供完整 8 类。

分类总览底部的训练系统参考不再使用页面内展开卡，也不再包在额外外层卡片中。默认只展示标题、摘要、关键阶段标签和 `查看` 动作；点击后打开底部半弹窗显示完整步骤流：

- `学习地图`：入门基础、身体与声音、角色关系、叙事结构、格式与主理。
- `训练路径`：个人热身、双人场景、小组排练、演出准备、演后复盘。

路径半弹窗支持查看和编辑我的版本。默认预设保持只读；用户首次编辑时保存为当前用户自己的 `路径` 素材副本，之后再次编辑更新该副本。个人副本使用 `relatedMaterialId` 关联预设 key，仍保持 `referenceOnly: true`，不进入抽卡训练池、训练计时或排练计划候选。

筛选半弹窗统一承载：

- 素材类型
- 训练能力
- 场景
- 状态

在类别详情态中，素材类型由当前类别锁定，筛选弹层不重复展示素材类型。

`路径` 属于低频参考型素材，只支持查看、收藏和筛选跳转；随机抽卡只从非路径素材中抽取。

## 4. 弹层矩阵

当前产品使用半弹窗和悬浮卡片承载轻量交互：

- 筛选素材：素材类型、训练能力、场景、状态；类别详情态中隐藏素材类型。
- 条件抽卡：抽中素材、再抽一张、进入详情。
- 添加素材：名称、素材类型、人数/时长、标签、简短描述、可选训练步骤。
- 学习地图 / 训练路径：查看完整路径步骤；编辑时生成或更新当前用户的个人路径副本。
- 快速开始练习：搜索素材、默认展示可选素材列表、选择后开始单次练习。
- 快速开启排练：团队、时长、目标、添加素材方式。
- 关联选择：选择关联素材或关联排练。
- 添加到排练：搜索素材并加入排练。
- 今日记录：查看今日灵感和今日排练。
- 个人沉淀/灵感列表：在我的页查看列表。
- 记录详情悬浮卡：查看单条沉淀或灵感详情。

## 5. 页面与弹层使用规则

- 主 Tab 承载高频入口和聚合状态，不承载过长表单。
- 子页面承载深读、完整编辑和排练过程。
- 轻量半弹窗承载筛选、抽卡和关联选择，建议高度控制在 50-65vh。
- 任务型半弹窗承载快速添加素材、快速开启排练等单一临场任务，建议高度控制在 70-82vh。
- 超过一个主任务、需要长表单或完整编辑时，应进入子页面。
- 任务型半弹窗的主按钮不强制固定在独立底栏；当独立底栏造成遮挡或视觉断层时，应回到内容流末尾。
- 悬浮卡片承载当前页面内的只读详情，不再额外增加跳转深度。
- 临场记录链路应尽量控制在入口 -> 记录 -> 保存。

## 6. 关键链路

### 6.1 找素材

发现页 -> 搜索/筛选/分类 -> 素材卡片 -> 素材详情 -> 开始训练。

详情页用于深读判断和启动单次练习，发现页卡片和随机抽卡不直接进入练习复盘。

详情页同时承载当前用户素材的完整编辑和删除：用户创建的素材或自己的路径副本显示编辑入口，系统公共素材保持只读；删除只从当前用户素材库软删除，不影响公共素材。

当前阶段补充：

- 无素材库时，发现页进入单一空态，主动作是 `添加素材`。
- 一旦存在至少一个素材，发现页恢复搜索、筛选、分类、最近使用和抽卡结构。

### 6.2 素材练习复盘

素材详情页点击开始训练 -> 暂停或结束复盘 -> 练习复盘页 -> 选择“保存记录”或“保存并沉淀为方法卡”。

目标体验是保留两条并列出口：

- 只保存当前练习记录，直接完成。
- 保存当前练习记录，并继续沉淀为方法卡。

补充边界：

- `路径` 素材不进入训练计时和练习复盘。
- 历史排练保持只读，不允许从练习复盘页回写或追加编辑。
- 素材、灵感、排练复盘的后续归档统一走“我的 -> 待整理”。
- 素材详情页底部应展示“个人练习记录”摘要，包含平均分、训练次数、最近 3 条记录和 `查看全部`；该模块只展示当前素材的练习记录，不混入其他素材。用户上滑只是自然滚动看到记录区，不作为隐藏跳转手势。
- 练习记录页承接完整筛选、详情和附件回看；素材筛选使用显式筛选入口和底部选择面板承载，不使用隐式系统 picker，也不把所有素材展开成长横向 chip。面板内固定提供“全部素材”、素材名称搜索和当前选中态；素材候选只列出已经有练习记录的素材，无记录素材不作为筛选候选。评分和附件类型这类短选项可继续用少量 chip。单条记录详情优先使用当前页底部弹层或详情面板，避免继续加深页面层级。
- 纯手势“上滑进入新页面 / 下滑返回详情页”不进入当前个人版；所有跨页面路径必须有可见按钮或列表项。

### 6.3 灵感记录

记录页 -> 文本记录灵感 -> 保存灵感 -> 我的页待整理或灵感记录 -> 可先选择训练线索 / 排练线索，再沉淀为方法卡并保存用途。

灵感强调“先别忘”，不应强制一开始就补齐所有结构化字段。

训练线索和排练线索是灵感的本地用途标记，只用于灵感记录列表筛选回看，不创建练习、排练或云端关联。

### 6.4 排练过程

记录页 -> 快速开启排练 -> 排练记录页 -> 添加素材、切换状态、记录 Keep / Try -> 暂停或结束复盘。

排练记录页主要处理过程态，不承载完整整体复盘。

### 6.5 排练复盘

排练记录页 -> 排练复盘页 -> 保存整体 Keep / Try / 下次提醒 -> 完成。

复盘保存后应尽量直接完成，不再强制多次确认。

### 6.6 资产回看

我的页 -> 待整理 -> 悬浮详情卡 -> 可临时选择训练线索或排练线索 -> 不再整理时忽略选择 / 沉淀为方法卡成功后保存选择。

我的页 -> 个人沉淀 -> 来源筛选 -> 悬浮详情卡。

我的页 -> 灵感记录列表 -> 按全部、待整理、训练线索、排练线索、已沉淀筛选 -> 悬浮详情卡。

我的页 -> 练习记录 -> 练习记录列表。

我的页 -> 排练记录 -> 当前页只读排练详情。当前个人版中的排练是个人多素材 session，不表达团队协作。

## 7. 编辑态与查看态

当前排练记录页承担过程编辑；历史排练回看已先收敛到 `team-records` 当前页的只读详情态。后续如果只读详情继续变复杂，再拆独立详情页。

- 排练过程页：可编辑、可暂停、可结束。
- 当前排练详情：`team-records` 页内只读回看，产品口径为个人多素材 session。
- 后续规划：如果需要复制为新排练、深链接直达或承载更多详情，再拆独立排练详情页。

历史排练回看不应默认进入可编辑排练过程页，避免查看历史时误操作。团队空间、成员权限、多人共同编辑和团队统计归入后续大版本。

## 8. 深度控制目标

目标链路：

- 发现 -> 详情 -> 开始训练 -> 结束复盘 -> 保存反馈。
- 发现 -> 抽卡 -> 详情 -> 开始训练 -> 结束复盘 -> 保存反馈。
- 记录 -> 文本输入 -> 保存灵感。
- 排练记录 -> 结束复盘 -> 保存完成。

避免链路：

- 发现 -> 详情 -> 反馈 -> 确认 -> 方法卡。
- 记录 -> 灵感页 -> 多字段填写 -> 保存前确认。
- 排练记录 -> 复盘 -> 确认 -> 方法卡。

## 9. 当前阶段数据展示约定

- 当前功能开发和页面交互开发阶段，不启用 `store` 本地持久化缓存。
- 页面只使用：
  - 云端真实数据
  - 当前会话内存态
  - 空态 / 错误态
- 如果之前运行过旧版本，验证空态前需要先清理微信开发者工具 Storage / 清缓存 / 编译缓存。
## 第三部分：体验与交互规范

更新时间：2026-06-19

## 1. 体验原则

即兴工具箱是临场操作工具，不是纯内容浏览 App。设计优先级应围绕：

- 现场记录链路要短。
- 保存动作要直接。
- 整理沉淀要后置。
- 高频操作尽量靠下，适合单手完成。
- 子页面只承载深读、完整编辑和排练过程。
- 半弹窗承载轻量选择、快速输入和局部确认。

### 1.1 发现页浏览规范

发现页默认服务临场速查，有素材时默认进入 `全部` 模式，展示搜索、新建、筛选和素材列表。`分类` 是第二模式，用于浏览素材体系；分类网格不展示搜索框，点击类别后进入类别详情态，再展示类别内搜索和列表。发现页不使用 `列表 / 卡片` 这类纯展示样式切换，避免用户以为只是视觉样式变化而不是任务变化。

发现页 `全部 / 分类` 模式切换属于页面主导航状态，选中态统一使用品牌浅底和品牌文字；标题区的 `新建` 使用轻量品牌按钮，`分享 / 筛选` 使用中性工具按钮。蓝色只用于筛选结果、信息提示等辅助语义，不与品牌橙同时争夺页面主操作层级。

发现页工具栏按钮保持 `分享 / 新建 / 筛选` 同排布局。`新建` 只使用低饱和暖橙浅底和深橙文字，不使用实橙、重阴影或主按钮视觉；`分享 / 筛选` 使用稳定浅蓝灰底、灰蓝文字和清晰边界，避免在白底上过淡。

## 2. 配色语义

页面和组件配色统一从 `frontend/styles/theme.wxss` 的 `--improv-*` token 获取，不在局部 WXSS 中散落硬编码色值。

- **主题切换**：当前提供“默认”与“沉浸（theme-vivid）”两套主题。所有页面根节点（`.page`）、TabBar 和半弹窗等组件，必须能接收并正确渲染当前主题变量。
- 主动作、保存、添加等高强调入口使用 `primary` 橙色/玫瑰色体系。
- 筛选选中、次级入口等辅助强调使用 `secondary` 体系；收藏爱心使用独立的 `favorite` token（当前默认主题为奶蓝灰未收藏、奶杏粉已收藏），避免把收藏状态硬编码在组件局部。
- 页面背景、卡片、表单输入和弹层使用 `bg`、`card`、`sheet`、`field` 表面 token，**坚决避免在半弹窗或卡片中硬编码纯白色（#ffffff 或 #fff）**，以免在切换主题时出现割裂。
- 正文、弱文本、辅助说明和分割线使用 `text`、`muted`、`light`、`line` 中性色 token。
- 云端写入失败使用 `error` 或 toast error token；`warm` 只用于非阻断注意状态。
- 渐变、阴影、遮罩和条纹也必须使用 token，避免只抽纯色而留下不可替换的品牌色。

### 2.1 卡片角色分级

同一页面中的卡片必须先区分角色，再决定装饰强度；不要让所有卡片都落成同级白底阴影块。

- `导航入口卡`：用于分类入口、快捷入口，视觉应更轻，更像目录，不承担长摘要。
- `内容摘要卡`：用于素材、记录等信息预览，重点在标题、摘要、meta 和状态节奏。
- `流程参考入口`：用于学习地图、训练路径等认知地图，默认只展示摘要和关键阶段，完整步骤进入半弹窗查看；如允许编辑，应编辑当前用户自己的路径副本，不直接修改官方预设。
- `容器操作卡`：用于表单、选择容器和详情浮层，重点是内容编排和操作分组，不与导航卡争主视觉。

实现要求：

- 页面优先通过语义 token 和卡片角色 class 表达层级，不在单页中重新发明阴影、圆角和边框体系。
- 代码层面优先使用 `card-surface-nav`、`card-surface-content`、`card-surface-action`、`card-surface-feedback`、`empty-state-card`、`state-panel-*` 等语义 class 表达角色，不用页面私有色值制造同级卡片变体。
- 同一屏中主导航区与辅助参考区必须明显分层，避免“入口卡”和“说明卡”视觉撞脸。
- 数量 badge、kicker、meta、选中态都应跟随卡种角色收敛，不能各组件自成体系。

### 2.2 按钮层级

按钮样式必须先按动作语义选择全局类，再由页面补充必要的布局定位；不要为同类动作在单页重新定义独立胶囊样式。按钮颜色统一来自 `frontend/styles/theme.wxss` 的 `--improv-btn-*` token，默认主题以品牌橙为主，沉浸主题通过 token 覆盖，不在页面局部写死橙色、蓝色、白底描边或阴影。

- `.primary-btn`：主动作，用于保存、提交、开始训练、开始排练、结束并复盘、查看结果、加入素材库。每个任务容器最多一个主动作，使用品牌实底或品牌渐变。
- `.ghost-btn`：次主动作，用于取消、再抽一张、沉淀入口、清空条件等与主动作并列但强调更低的操作。默认是品牌浅底，不再使用蓝底作为通用强调。
- `.small-btn` / `.button-tertiary`：卡片右上角、列表项和详情卡中的低强调动作，例如编辑、整理、查看、选择、继续。默认是安静浅底或透明感按钮，不使用重阴影白底胶囊。
- `.mini-action-btn.neutral`：页面工具栏中的分享、筛选、搜索区清空等中性工具动作。
- `.mini-action-btn.primary-lite`：页面工具栏中的轻量新建、添加等小主动作，只用品牌浅底，不使用主按钮实底。
- `.chip` / `.compact-filter-chip`：分类、筛选、目标、状态、标签、效果反馈等选择态。inactive 使用中性浅底，active 使用品牌 tint，只表达选择状态，不承担命令按钮职责。
- `.icon-text + .icon-*`：返回、关闭、收藏、搜索等图标动作。随机悬浮球使用明确文字标签，是独立浮动动作，不参与普通按钮层级。
- `.button-danger-quiet` / `.danger-btn`：删除、不再整理、拒绝授权等危险动作。默认使用危险浅底或危险文字，不使用实底红；最终确认可继续使用系统确认或单独确认弹层。

同一页面中同级按钮必须保持一致的高度、圆角、边框、字号和 token 来源。局部 WXSS 只允许控制宽度、定位和排列，不应重新覆盖背景、边框、阴影和颜色体系。

### 2.3 图标语义

图标统一采用文本或内联符号时，必须先使用 `.icon-text`，再按含义追加 `.icon-back`、`.icon-close`、`.icon-search`、`.icon-random`、`.icon-favorite` 等语义类。页面 WXSS 只补充布局位置，不重新定义颜色、阴影和点击热区。新增图标含义时，先扩展全局语义类，再在页面引用。

## 3. 临场记录规范

### 3.1 练习复盘

默认首屏优先展示：

- 记录对象
- 归属场景
- 效果反馈
- Keep
- Try
- 保存记录
- 保存并沉淀为方法卡

人数、下次提醒等字段可作为更多信息，不应挤占临场记录首屏。

如果用户点击“保存并沉淀为方法卡”，页面应先落下完整的练习复盘记录，再继续创建方法卡；不要出现“方法卡已生成，但练习记录没保存”的断链。

### 3.2 灵感记录

灵感强调“先别忘”，允许粗糙和未整理。

记录页优先直接展示文本输入框，保存后形成灵感草稿；灵感编辑页用于稍后整理，而不是保存前必经页。当前版本不提供 app 内置音频采集、转写或自动摘要，口述输入交给系统输入法自带的语音转文字能力。

### 3.3 排练记录

排练记录页主要处理过程态：

- 素材计划。
- 素材状态。
- 每个素材 Keep / Try。
- 添加素材。
- 暂停和继续。

整体复盘放到排练复盘页，避免排练过程中页面过重。

## 4. 弹层规范

半弹窗属于模态底部面板，应满足：

- 背景变暗。
- 背景不可点击。
- 背景不可滚动。
- 焦点切换到弹层。
- 底部视觉由弹层接管。

半弹窗按用途分为三类：

- 轻量半弹窗：筛选、抽卡、关联选择，承载轻选择和局部确认，建议高度控制在 50-65vh。
- 任务型半弹窗：快速添加素材、快速开启排练，只承载一个临场任务，建议高度控制在 70-82vh。
- 子页面替代：完整编辑、深读、长表单、排练过程和复盘，不应继续塞进半弹窗。

任务型半弹窗必须保持标题区稳定，内容区与底部操作区独立。提交、保存、开始和确认等页面级主操作统一放入组件底部操作区，底栏使用弹层同一表面色并保持紧凑；列表条目自身的选择、加入等行级动作仍留在条目右侧。半弹窗默认采用 `fit` 正文模式，由普通容器按实际内容收缩并设置最大高度，超出上限后只滚动正文；仅确需固定原生滚动视口的场景显式使用 `scroll` 模式，并为 Skyline 的纵向 `scroll-view` 设置明确高度。两种模式都必须保证底部主操作始终可见。

任务型半弹窗必须显式设置 `closeOnMask="{{false}}"`，避免误点遮罩导致表单、路径编辑、个人资料、方法卡或排练计划调整丢失。轻量筛选、抽卡、列表查看和只读详情弹层可以保留遮罩关闭。

任务型选择弹层应优先使用“搜索框 + 默认可见列表 + 确认按钮”结构，而不是把“选择”做成输入框旁边的窄按钮。记录页快速开始训练、排练中添加素材这类场景，打开后应直接展示可选项；搜索只作为缩小范围的辅助能力。列表必须在弹层内部独立滚动，并保留稳定高度，确保小屏设备上主按钮和底部安全区不被遮挡。

对于“添加到当前计划 / 当前上下文”一类选择弹层，列表只展示当前仍可执行的候选项，不重复展示已经加入、已经关联或当前不可再次操作的对象；如果可选项为空，应直接给出原因和下一步动作，避免用户点击后收到假成功反馈。

“添加素材”半弹窗只做快速收录：名称、类型、能力和场景为核心字段，标签、人数、时长和简短描述为推荐填写，玩法步骤默认作为可选补充项。字段必须分成 `类型`、`能力`、`场景`、`标签` 四个语义：类型必选且单选，只能使用固定 8 类；能力和场景可多选；标签可自由添加和删除，但不创建新的一级分类。表单布局必须遵守亲密性原则：字段标题与对应输入框或选项行靠近，不同字段组之间用更大的垂直间距分开。顶部提示文案应收敛，保存按钮放在底部操作区并居中显示。

素材详情页的编辑态应复用添加素材的字段分组和视觉节奏，但只面向当前用户创建的素材或当前用户的路径副本；系统公共素材只读，不显示编辑或删除入口。删除属于危险操作，必须先确认，并在云端成功后再从当前列表移除。

学习地图、训练路径这类低频参考内容应作为轻量入口进入半弹窗，不在主页面内点击展开。半弹窗查看态展示标题、摘要、能力标签、阶段步骤和备注；编辑态只保留标题、描述、阶段步骤、能力标签和备注等最小字段。自定义采用“个人副本”模式：首次保存创建当前用户自己的 `路径` 素材，后续保存更新该副本；路径副本仍为 `referenceOnly`，不进入训练计时、抽卡或排练计划候选。

半弹窗内复用筛选、表单字段、卡片等组件时，父页面只负责组件之间的垂直节奏，组件内部的标题、滚动行、chip 高度和输入框间距必须在组件自身样式里定义。不要在页面 wxss 里用跨组件深层选择器修子组件内部布局，以免 Skyline / Glass-Easel 组件隔离下样式不生效或在不同弹层里表现不一致。

所有表单字段默认使用文本输入，不提供 app 内语音按钮。对多行 textarea，优先保留完整编辑空间、文本可读性和滚动区域。

筛选弹层必须按分类分行展示，不使用所有 chips 混排的通用网格。当前筛选分类为人数、时长、目标和状态。每一类独占一行，分类标签在左，选项在右侧横向排列并允许换行。这样用户先理解筛选维度，再选择条件，避免不同维度互相挤压和视觉混乱。

实现优先级：

1. 官方模态容器能力。
2. `root-portal`。
3. 自制 fixed 遮罩。

不建议继续在各页面中堆局部样式补丁。

## 5. 底部操作栏规范

底部固定操作栏常见于详情页和表单页，必须统一处理：

- 操作栏高度稳定。
- 固定在屏幕底部的主操作区，统一距离屏幕下边缘 `24rpx + env(safe-area-inset-bottom)`。
- 内容区底部 spacer 等于操作栏高度 + 下边缘距离 + 安全区 + 至少 `24rpx` 内容保护距离。
- 半弹窗自身统一距离屏幕下边缘 `16rpx + env(safe-area-inset-bottom)`；半弹窗内固定底部按钮距离弹窗视觉底边 `24rpx`。
- 不出现超大空白。
- 不用大块纯白硬遮挡覆盖页面内容。
- 表单最后一个输入项必须能滚动到操作栏上方至少 24rpx。
- 主 Tab 页面底部预留应与自定义 TabBar 统一。

推荐将通用样式抽成 `fixed-action-bar` 或共享样式，而不是每页单独写。

## 6. 顶部安全区与返回按钮

微信小程序页面必须以真实胶囊位置为布局基准，而不是写死顶部 rpx：

- 返回按钮与微信右上角胶囊在同一行。
- 返回按钮只保留箭头，不显示“返回”文字。
- 返回按钮左侧应与页面内容左边界对齐。
- 除返回按钮外，子页面标题、卡片和正文顶部不能进入胶囊所在行。
- 主页面内容从胶囊行下方开始，不额外叠加一整段导航空白。

实现上通过全局布局变量承接：

- `--nav-row-top`
- `--nav-row-height`
- `--content-safe-top`
- `--page-side`
- `--page-bottom`

页面根节点必须绑定 `layoutStyle`，避免不同页面各自猜测安全区。

## 7. loading / empty / error

所有异步页面和列表至少考虑三态：

- loading：请求中展示轻量加载态，不阻塞已可用内容。
- empty：没有数据时展示下一步动作，例如添加素材、记录灵感、开启排练。
- error：服务失败时展示可理解提示，并提供重试。

实现上优先复用 `empty-state-panel`，并通过 `tone=loading/error/empty/notice` 表达状态语义；不再为 loading、empty、error 在页面局部各写一套独立卡片。

当前阶段补充约定：

- 当前功能开发阶段不启用业务本地持久化缓存，页面的历史业务数据只使用云端确认结果。
- 云端成功返回空数组时，必须进入 empty，不允许再回退旧本地数据。
- 云端失败时进入 error；读取失败不得降级为 empty。
- `mine`、`record`、`team-records` 这类聚合页，要明确区分“空数据”和“请求失败”。
- 首次无数据且页面目标明确时，应优先采用单一主动作空态，不保留多个需要数据支撑的竞争入口。
- 首次进入时，身份建立不应抢在核心任务之前；登录或授权提示应尽量绑定到真实保存、同步和回看动作。

CloudBase 写入失败时应保留表单草稿；如后续需要离线能力，必须同时设计持久化队列、幂等键、重试和冲突处理，不再使用内存标记代替。

`mine` 页中“已不再整理”属于用户的本地整理偏好，而不是业务记录本身。它可以在本地记住，避免用户每次重开小程序都重新忽略同一条待整理项；但原始灵感、练习记录和排练复盘仍保持不删除、不回写云端。

### 7.1 极简展示规则

在当前产品定位下，页面的“简洁和优雅”主要通过信息减法实现，而不是增加装饰层。

- 零数据模块默认隐藏，不因页面平衡而强行保留 `0 条` 卡片或空白容器。
- 正式资产卡只展示已经发生的数据；尚未发生的内容更适合收敛为一张轻引导卡。
- 数字为 `0` 的统计，不默认占据视觉主位；只有进入真实积累阶段后，统计才提升为概览信息。
- 空态优先使用页面内轻卡片承接，包含标题、解释和一个下一步动作，不优先依赖 Toast 短路。
- 首屏优先保留一个主任务块和一个次级上下文块；没有真实价值的推荐、关联、扩展信息默认后置。
- 可选关联、筛选和补充说明在没有真实数据前应折叠、隐藏或弱化，不拉长首屏。
- 同一个页面中，空数据与失败状态必须视觉区分：空数据给下一步，失败给重试。

## 8. 保存与沉淀

保存和沉淀是两类动作：

- 保存：把当前记录先留下来，动作应直接。
- 沉淀：把记录整理成可复用方法卡，属于二级动作。

不应在每次保存前强制用户确认是否沉淀。更好的方式是：

- 保存成功后提供“沉淀为方法卡”的入口。
- 在我的页集中整理待沉淀记录。
- 对高价值记录给出轻提示，而不是中断式弹窗。

待整理中的灵感除了不再整理和沉淀为方法卡，还可以临时选择训练线索或排练线索；选择本身不产生保存或关闭动作，沉淀成功后才保存用途，不再整理则忽略选择。线索只是未来用途标记，用于在灵感记录中筛选回看；它不应在当前版本里自动创建练习、创建排练或回写历史排练。

## 9. 轻提示规范

轻提示用于反馈低成本操作，不应遮挡正在填写的字段：

- 普通成功提示使用柔和白色半透明胶囊，例如 `已保存灵感`、`已加入素材库`。
- 保存失败使用 error 语义，不使用暖色成功胶囊。
- 轻提示背景、边框、文字和阴影使用 `theme.wxss` 中的 toast token，不在页面局部重写色值。
- 不使用黑色系统 toast 作为常规反馈；系统 toast 只作为拿不到当前页面实例时的兜底。
- 轻提示通过全局 `toast(title)` 触发，页面不单独创建局部提示层。
- 轻提示只承接操作结果，不承担保存确认流程。

## 10. 页面深度目标

推荐目标：

- 找素材并开始：发现 -> 详情 -> 开始训练 -> 结束复盘 -> 保存反馈。
- 抽卡后开始：发现 -> 抽卡 -> 详情 -> 开始训练 -> 结束复盘 -> 保存反馈。
- 灵感记录：记录 -> 文本输入 -> 保存灵感。
- 排练复盘：排练记录 -> 结束复盘 -> 保存完成。

超过两步的临场动作，应优先考虑半弹窗或保存后整理，而不是继续增加子页面。

## 11. UI 体系维护与审计

每个页面和组件变更必须先确定语义，再选择实现：

- 卡片只使用导航、内容、参考、操作、反馈、容器六类角色；需要新角色时先更新架构和体验规范。
- 新增颜色、阴影、圆角和字号前，先确认现有主题 token 和层级无法表达；Skyline fallback 必须与对应 `var(--improv-*)` 同行。
- 新列表必须显式覆盖 loading、empty、error；写入成功必须以服务端响应为准。
- 普通文本表单使用 `bindblur` / `bindconfirm`。若确需 `bindinput`，必须先完成 Skyline 真机中文组合输入专项验证。
- 任务型半弹窗显式设置 `closeOnMask="{{false}}"`；轻量筛选、抽卡、关联选择和只读详情可保留遮罩关闭。
- 手势不能成为唯一操作路径；左滑删除、翻页和拖动必须有可见按钮或说明作为替代。

### 11.1 移动端布局防溢出规则

卡片、半弹窗和列表项必须先保证内容在 iPhone 12/13 等窄屏宽度内不越界，再考虑视觉装饰：

- 卡片内横向布局必须设置 `box-sizing: border-box` 和 `min-width: 0`，长标题、素材名和按钮文案使用单行省略，不允许撑破卡片。
- 卡片内双按钮默认允许换行或等分收缩；不得用固定 `min-width` 让两个按钮总宽超过卡片内容区。
- `empty-state-panel` 作为空态基础设施，动作区必须由组件自身保证不溢出，调用页不得用页面级深层选择器修补内部按钮。
- 底部选择面板中的列表项必须全宽、左对齐，并显式覆盖原生按钮居中；选择态图标只作为右侧状态，不制造无意义空列。
- 混合字号徽标，例如 `7 分`，必须垂直居中并使用稳定行高；只有明确的排版理由才允许使用 baseline 对齐。
- 子页面头部只表达页面身份；素材名、练习对象和关联方式放在内容卡片里。时长、状态等过程数据需要突出时，应使用独立摘要块，不与页面标题互相重复。

较大 UI 迭代结束后复算组件引用、未绑定主题的直接色值、字号/圆角/间距档位、页面私有状态卡、`bindinput` 和任务型弹层配置。量化基线和当前整改清单以 [ui-architecture-audit.md](ui-architecture-audit.md) 为准。

## 第四部分：Zentro 双主题设计契约

### 1. 主题名称与兼容

- 设置页只展示 `灵感` 和 `现场` 两个选项。
- 现有持久化值保持 `default` / `vivid`；无记录时使用 `default`，不做数据迁移。
- `default` 映射品牌浅色“灵感”，`vivid` 映射品牌深色“现场”。
- `theme-alt` 不对用户开放，作为废弃候选等待实施阶段确认无引用后移除。

完整色值和 Logo 边界以 [品牌视觉规范](brand-guidelines.md) 为准。产品端只消费语义 token，不直接消费“橙/粉/紫/蓝”等原始色名。

### 2. 主题使用场景

- 灵感：白天、日常浏览、快速记录和长文本编辑；使用近白暖灰到极浅暖黄/浅青灰背景、纯白高明度卡片、深灰黑正文、灰蓝辅助层级，主操作采用明亮橙黄配深棕文字，工具蓝承担链接与 Tab 选中，活力绿只用于成功、进度和正向点缀。
- 现场：排练、演出后台和低光环境；使用深墨/深灰紫表面、浅色文字，紫色作为主操作，蓝/橙/粉承担信息与状态。
- 两套主题的布局、组件、字号、触控目标、内容顺序和交互行为完全一致；不得为主题复制 WXML 或页面 WXSS。

### 3. 栅格、间距和触控

- 页面以 750rpx 设计宽度为基线，左右安全边距使用既有 `--page-side`，不得用局部负边距突破系统安全区。
- 间距基线为 4/8rpx；常用语义步长为 8、12、16、24、32、48rpx。
- 主操作、Tab 和关键图标的点击区域目标不小于约 88rpx；视觉图标可以更小，但热区不得缩小。
- 卡片和字段在窄屏自然纵向排列；横向 chip 允许滚动或换行，不缩小文字到不可读尺寸。

### 4. 字体、图标和动效

- 中文使用思源黑体方向并回退 PingFang SC；英文数字使用 Poppins 方向并回退系统无衬线。
- 返回、关闭、搜索、收藏、随机和分享最终使用同一图标来源；手势必须提供可见替代入口。
- 动效只保留 fast 160ms、normal 200ms、slow 300ms 三档；pressed 使用 0.98 缩放，弹层使用 200–300ms 位移与淡入。
- 系统开启减少动态或设备性能不足时，取消非必要位移与缩放，保留即时状态反馈。
- 不使用循环装饰动画，不让品牌渐变持续流动干扰临场操作。

### 5. 组件状态矩阵

| 组件 | 必备状态 | 双主题验收重点 |
| --- | --- | --- |
| 按钮 | normal/pressed/disabled/loading | 主次层级、4.5:1 文字对比、防重复提交 |
| 输入 | normal/focus/filled/disabled/error | 标签持续可见、错误有文字、输入表面层级 |
| chip/选择器 | normal/selected/disabled | 颜色之外同时改变边界和文字 |
| 卡片 | static/pressed/selected/loading/error | 六种卡片角色不因主题改变职责 |
| bottom-sheet | opening/open/submitting/error/closing | 遮罩、正文滚动、操作区、草稿和焦点 |
| TabBar | inactive/active/pressed/hidden | 当前采用纯文字 Tab，不使用无语义装饰圆点；active 使用整块加深的暖橙胶囊背景、文字和边界同时表达，inactive 保持可读灰蓝，安全区稳定 |
| 状态面板 | loading/empty/error/notice | 空态给下一步，错误给重试，不互相伪装 |

### 6. 无障碍底线

- 普通文本对比度至少 4.5:1，大字至少 3:1；品牌原色不因“视觉统一”豁免。
- `#FF6A00` 不直接承载普通字号白字；使用深墨文字，或采用品牌批准的深橙派生色。
- 原始 `#0070FF` 配白字不足 4.5:1 时，按钮使用更深蓝派生色。
- success/warning/error、selected/disabled 不得只依赖颜色。
- 截图只能证明视觉风险；读屏、焦点顺序、真机中文输入、动态和安全区必须真机验证。

### 7. 设计变更门禁

1. 需求评审先确认现有组件和语义 token 是否可表达。
2. 视觉稿同时给出灵感/现场两套关键状态，不只给静态默认态。
3. 实现后检查直接色值、触控目标、三态、提交防重、弹层关闭和减少动态。
4. 微信开发者工具完成全页面截图回归；中文输入、手势和安全区使用真机。
5. 发布后按任务完成率、耗时、失败保稿率和重复提交率复盘，不以主观“更好看”代替指标。

本文档梳理当前项目中已经统计并整理为可导入数据的全部素材资料。正式可导入数据以 `mock_data/improv_materials.json` 和 `mock_data/improv_materials.import.json` 为准。

## 1. 结论摘要

- 当前完整预置素材共 163 条，全部面向 `improv_materials` 集合。
- 公共素材统一使用 `ownerOpenId: "system"`，无需替换用户 openId。
- `mock_data/improv_materials.json` 是标准 JSON 数组，适合人工审阅和脚本处理。
- `mock_data/improv_materials.import.json` 是 JSON Lines 内容但保留 `.json` 后缀，适合微信开发者工具数据库导入窗口。
- `路径` 类素材全部为 `referenceOnly: true`，只用于查看、收藏和筛选跳转，不进入训练计时、抽卡训练池或排练计划。

## 5. 导入数据字段

每条素材记录必须包含并已包含以下字段：

- `id`：素材业务 ID，当前 163 条均唯一。
- `title`、`desc`：素材标题和一句话说明。
- `type`：`游戏`、`角色`、`才艺`、`格式`、`主理`、`技巧`、`复盘`、`路径` 之一。
- `tags`、`abilities`、`scenes`、`meta`、`steps`：数组字段，用于筛选、检索和详情展示。
- `tips`、`variant`、`issue`：补充提示、变体和风险提醒。
- `relatedMaterialId`：关联素材 ID；当前预置素材为空字符串。
- `referenceOnly`：路径类为 `true`，其他类型为 `false`。
- `stripeTone`：`orange`、`blue`、`mint` 之一。
- `sortOrder`：发现页排序权重。
- `ownerOpenId`：公共素材固定为 `system`。
- `createdAt`、`updatedAt`、`deletedAt`：导入时间和软删除字段。

## 6. CloudBase 手动导入建议

1. 在 CloudBase 控制台确认已创建 `improv_materials` 集合。
2. 使用微信开发者工具或 CloudBase 控制台导入 `mock_data/improv_materials.import.json`。
3. 导入格式选择 JSON Lines / 一行一条记录。若工具只允许 `.json` 后缀，仍使用该文件。
4. 导入后调用 `material.list` 验证发现页素材；如页面仍显示旧数据，清理微信开发者工具 Storage、清缓存和编译缓存。
5. 私有样例集合导入前仍需替换 `__REPLACE_WITH_OPENID__`，但 `improv_materials` 不需要替换。

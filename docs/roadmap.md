# 产品路线与优先级

更新时间：2026-06-07

## 1. 当前阶段判断

当前产品方向成立：它解决的是即兴排练现场的“找游戏、快记录、可沉淀”问题。

主要风险不在功能不足，而在：

- 记录链路偏深。
- 子页面和弹窗职责边界不够清晰。
- 底部 fixed 操作栏和滚动区域缺少统一规范。
- 状态层已收敛到轻量内存 store，后续重点是逐步把旧 JS 页面迁移到 TS。

## 2. P0：发布前阻塞

- 已收敛：游戏反馈页底部白色遮挡和内容 spacer 不一致问题。
- 已收敛：fixed 操作栏样式和底部安全区策略，子页面返回箭头与胶囊同一行。
- 已收敛：保存反馈直接保存，移除强制记录确认弹窗。
- 已收敛：全页面常规提示改为统一轻提示，不再使用黑色系统 toast 作为常规反馈。
- 已收敛：CloudBase `improv-api` action、前端服务层和工程文档已静态核对对齐。
- 已收敛：`improv_` 集合初始化、权限和 seed 流程已在 `wechat-cloudbase-app/database.md` 和 `docs/data-api.md` 中对齐。

## 3. P1：降低临场记录成本

- 已收敛：语音速记完成后可直接保存灵感草稿。
- 已收敛：游戏反馈页精简首屏字段，把人数、关联排练和下次提醒折叠到更多信息。
- 已收敛：排练复盘保存后直接完成。
- 已收敛：我的页集中承接“待整理”和“沉淀为方法卡”。

## 4. P2：信息架构收敛

- 已收敛：团队排练记录进入只读详情，而不是默认进入可编辑排练过程页。
- 已收敛：筛选弹层按人数、时长、目标、状态分行展示，不再混排。
- 已收敛：方法卡支持按来源筛选，并在待整理入口中标记来源类型。
- 已收敛当前阶段：已删除未引用的旧 `local-state.js`，页面状态来源统一回到 `store/index.ts` 和云端结果。

## 5. P3：增强能力

- 已收敛当前阶段：CloudBase 写入失败时保留当前会话 `pending` 状态，并在记录、我的、团队记录入口显示“待同步”。
- 已收敛当前阶段：团队记录页补充 loading、empty、error 三态；我的列表和今日记录弹层补充空态/待同步提示。
- 团队多人协作编辑。
- 更完整的语音转文字和摘要能力。
- 自动推荐游戏和排练模板。
- 方法卡分享。
- 高级统计报表。
- 多团队/空间权限。

## 6. MVP 发布差距

MVP 发布前建议至少确认：

- 游戏库浏览、搜索、筛选、收藏、玩过状态可用。
- 游戏详情和添加游戏可用。
- 灵感记录可用。
- 游戏实践反馈可用。
- 快速开启排练、排练游戏计划、游戏状态可用。
- 排练复盘可用。
- 我的页个人沉淀、灵感记录、排练记录可回看。
- CloudBase 数据隔离和错误提示可用。

## 7. 后置能力

以下能力不影响 MVP 基础验证，可后置：

- 多人共同编辑同一场排练。
- AI 自动生成完整排练模板。
- 方法卡公开分享和社区化。
- 高级团队统计。
- 复杂推荐算法。
- 多空间、多角色权限。
